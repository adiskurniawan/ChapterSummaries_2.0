Chapter 1 — Getting Started with Advance Excel
Overview & Purpose
Explains why advanced Excel matters for accounting & finance: faster analysis, fewer manual errors, reproducible models, better reporting. Describes workbook/table organization, naming conventions, and recommended best practices for auditability and reuse (separate raw data, calculations, outputs; use tables and named ranges; avoid hard-coded constants).

• How to convert a range to a Table (Ctrl+T).

• Table naming and why to use names (structured references in formulas).

Working with large data — performance tips
• Prefer tables and structured formulas over many volatile formulas.
• Avoid excessive use of volatile functions (NOW, TODAY, INDIRECT, OFFSET).
• Use helper columns for repeated logic rather than repeating complex formulas across columns.
• Prefer INDEX() over OFFSET() for dynamic ranges; limit use of array formulas unless needed.
Quick analysis & productivity shortcuts
• Useful shortcuts referenced: Ctrl+T (table), Alt+N+V (insert pivot), F4 (repeat/absolute), Ctrl+; (date), Ctrl+Shift+L (filters), Ctrl+Arrow (navigate).
• Use Quick Analysis tool (when available) for fast conditional formats, totals, charts.
Integration tips & final checklist
• Always validate source data, lock calculation cells, protect sheets with clear input areas only.
• Maintain version control: Save iterative copies, keep changelog sheet for major changes.
• Documentation: include a short 'Model Guide' worksheet listing assumptions, named ranges, key formulas, and scenario selectors.
• Common audit checks before sharing: reconcile totals to source, ensure no hidden sheets with sensitive data, run Formula Auditing, test scenario toggles.
Examples & hands-on exercises (recommended)
• Create a Table from raw transactions, add a Total row and structured SUMIFS metrics.
• Build a drop-down driven forecast: scenario selector → VLOOKUP/XLOOKUP to feed assumptions → summary pivot and pivot chart.
• Record a macro to format monthly report and convert it into a reusable sub.
Troubleshooting common user questions
• Lookups returning #N/A — check keys, trim, exact match flag.
• Pivot not showing updated rows — refresh or change source to Table.
• Conditional formatting not applying — check rule scope and rule order.
• Slow workbook — identify volatile formulas, reduce used range, convert formulas to values where final.
Security & sharing
• Protect sheets, use workbook protection cautiously (not a substitute for secure storage).
• For macros, use signed macros or trusted locations; document manual steps required for recipients.
Summary — key takeaways
• Structure data with Tables and Named Ranges.
• Validate and clean data before analysis.
• Prefer INDEX+MATCH/XLOOKUP for robust lookups.
• Use Pivot Tables and GETPIVOTDATA for repeatable reporting.
• Automate repetitious UI tasks with macros but keep them minimal, documented, and secure.
• Document assumptions and provide a model guide; perform reconciliation and sanity checks before distribution.

Chapter 2 — Preparing Financial Statements Smartly
Overview & Purpose
Explains why preparing financial statements matters for internal control, external reporting, decision-making, and auditability. Emphasises reproducibility, audit trail, clarity of presentation, and separation of raw data, calculation logic, and published outputs. Recommends a single-source-of-truth ledger (table) feeding standardized FS templates, with automated checks, clear mapping tables, and documented assumptions. Objective: produce Balance Sheet, Income Statement, Cash Flow Statement and Notes that are accurate, auditable, and easily refreshed for monthly/quarterly close cycles.
Financial statements covered
Balance Sheet (statement of financial position): classification, current vs non-current, working capital presentation.
Income Statement (profit & loss): revenue recognition basics, cost of goods sold, operating vs non-operating items, tax and extraordinary items.
Cash Flow Statement: indirect method (reconciles NI to CFO), investing and financing sections, capex schedules and loan flows.
Statement of Changes in Equity: share capital movements, dividends, other comprehensive income.
Notes & Disclosures: accounting policies, fixed-asset schedules, debt maturities, contingent liabilities, related-party transactions, subsequent events.
Design principle — single source, clear layers
Layer 1 — Raw source data: AP/AR sub-ledgers, bank statements, GL export, payroll file, fixed asset register, inventory movement reports. Avoid manual edits to this layer.
Layer 2 — Mapping & staging: mapping tables (GL → FS line), currency and period alignment, reclassification rules, adjustment ledger. Use tables and named ranges.
Layer 3 — Calculations: formula-driven schedules (depreciation, accruals, amortisation), rolling averages, turnover metrics. Keep all business logic here; no hard-coded ad-hoc edits.
Layer 4 — Presentation & published reports: formatted FS, management summary, variance commentary and KPIs. Only this layer is allowed formatting changes — never edit calculation cells directly.
Chart of accounts, mappings and metadata
Chart of Accounts (CoA) best practices: simple, hierarchical, stable GL codes; use natural account grouping (Revenue, COGS, OpEx, Financials, Taxes, Extraordinary). Maintain change log for any CoA edits.
Mapping table: tidy table that maps every GL code → FS line item + presentation group + consolidation tag + cashflow mapping; include currency, tax treatment and reclass flags.
Metadata fields: effective date, cost centre, segment, legal entity, consolidation scope and elimination flags to support pivoting and multi-dimension reporting.
Close & period process — recommended workflow
Pre-close: collect AR/AP ageing, bank confirmations, fixed asset movements, payroll, inventory counts.
Adjusting entries: post accruals, prepayments, depreciation, FX revaluation, tax provisions.
Reconciliations: reconcile GL ↔ subledgers (AR, AP, Inventory, FA, Bank) and prepare variance commentary.
Review: technical review (accounting), management review (material variances), CFO sign-off.
Publish: lock published FS; store as read-only with version tag and changelog.
Close checklist:
1. Export GL and subledgers.
2. Run automated mapping & initial trial balance.
3. Post standard recurring journals (depr, accruals, prepayments).
4. Reconcile key control accounts and fix recon differences.
5. Prepare draft FS and variance commentary.
6. Obtain sign-offs and publish.
Key adjusting entries — conceptual checklist
Accruals — expenses incurred but not invoiced; calculate from drivers or historical ratios and tie to subsequent invoices.
Prepayments — expenses paid but not consumed; amortise monthly with reversing schedule.
Depreciation & Amortisation — policy-driven (SLM or WDV), useful lives and commencement rules; link to FA roll-forward.
Inventory adjustments — reconcile perpetual vs physical; write-downs to NRV when required; ensure cut-off correctness.
Bad debt / allowance — ageing-based methodology with policy thresholds and approvals.
FX revaluation — apply functional currency policy and post unrealised FX gains/losses for monetary items.
Tax provisions — compute current tax and deferred tax from timing differences; maintain deferred tax schedule.
Excel implementation patterns for FS — practical rules
Tables & structured references: use Excel Tables (Ctrl+T) for source and staging data; reference via structured names.
Mapping tables & SUMIFS pattern: one stable mapping table and SUMIFS/SUMPRODUCT/INDEX-MATCH aggregation; avoid manual range sums.
Avoid direct external cell links — prefer Power Query or single workbook with separate sheets; document any external link refresh behaviour.
Helper columns vs array formulas: prefer helper columns for complex logic to improve transparency and performance.
Named ranges for drivers: centralise interest rates, tax rates, working days and other global parameters.
Automated checks and control totals — must-have validations
Trial balance check: Assets = Liabilities + Equity with automatic error highlighting.
Subsidiary reconciliations: AR total = AR sub-ledger; AP total = AP sub-ledger; Inventory GL = Inventory schedule.
Movement checks: reconcile cash movements to bank statements; verify capex roll-forward.
Reasonableness checks: margin percentages, EBITDA vs thresholds; flag large variances.
Control sheet: central 'Control & Exceptions' sheet listing checks, pass/fail, investigator and remediation note; use non-volatile status indicators where possible.
Cut-off, completeness and accuracy controls
Cut-off tests: verify sales & purchases around period end (last 7 days) to ensure correct period recognition.
Completeness: match bank transactions to GL; reconcile payments with AP ageing.
Accuracy: sample high-value invoices and trace to source; cross-check unit costs/quantities for inventory.
Cash flow statement — preparation pattern
Indirect method: Net Income → add back non-cash items (D&A, provisions) → adjust for working capital movements (ΔAR, ΔInv, ΔAP) → CFO.
Investing & Financing: capex from FA roll-forward, proceeds from disposals, loans drawn/repayments, interest and dividend flows.
Presentation: separate operating cash flows from interest and tax paid (disclose paid vs accrued); reconcile closing cash to bank statements.
Consolidation & intercompany basics
Scope & template: use entity-level staging sheets with identical columns for automated consolidation (Power Query / macros).
Eliminations: intercompany AR/AP netting, intercompany revenue/COGS elimination, remove unrealised profit in inventory when material.
Non-controlling interest: compute NCI based on ownership % and present NCI in equity and profit allocation.
FX for consolidation: translate BS using closing rates, P&L using average rates, and record translation reserve.
Notes, disclosures and supporting schedules
Fixed asset roll-forward: opening balances, additions, disposals, depreciation, closing balances; include capex policy and useful lives.
Debt schedule: facilities, rates, maturity profile, covenant terms and covenant compliance calculations.
Tax schedules: current tax, deferred tax movements and recon between accounting profit and taxable profit.
Related-party disclosures: list transactions and balances with policies.
Provisions & contingencies: movement schedules, expected timing and judgment notes.
Presentation & formatting for users and auditors
Readable layout: consistent font, thousand separators, negative numbers in parentheses, clear headings and subtotals.
Version control: naming convention (YYYYMMDD_Close_vX) and locked published copy.
Workpapers: include 'Model Guide' sheet listing data sources, named ranges, key formulas and contact owner.
Exporting: publish PDF only when needed; keep master workbook as audit file.
Audit readiness & traceability
Audit trail: every manual adjustment must reference supporting doc (invoice#, journal id), include preparer & approver fields in adjustment register.
Document storage: index or link supporting PDFs (bank confirms, contracts) stored in controlled repository with naming convention.
Sign-off logs: capture reviewer and final approval timestamps for key schedules.
Performance & reliability — speed & scaling tips
Minimise volatile formulas (NOW, INDIRECT, OFFSET) to reduce recalculation time.
Use Power Query for large GL imports and transformations and schedule refresh steps.
Aggregate before lookup: avoid many lookups across million-row ranges; aggregate source first.
Use INDEX/MATCH or XLOOKUP with exact match; consider Power Pivot/data model for heavy aggregation.
Reconciliations, exception handling & remediation workflow
Central reconciliation register: track status (open/cleared), owner, target date, root cause and remediation action.
Aging & investigation: automatically escalate open recon items above threshold to manager; retain clearance evidence.
Post-close review window: allow 5–10 days for discovering and correcting errors with documented adjustments and disclosure as needed.
Quality assurance & verification checklist (operationalised)
Top automated checks: TB balance; AR/AP subledger totals; cash bank match; FA roll-forward totals; debt schedule vs GL; tax provision reasonableness; major variance explanation present.
Reviewer tasks: sample invoice checks, confirm manual journals have approvals, sensitivity testing on key assumptions (FX, useful lives).
Verification protocol: run multiple independent checks across data, mapping, calculations and outputs before finalising (mix automated and manual reviews).
KPIs and management outputs
Core ratios: current ratio, quick ratio, inventory turnover, DSO, DPO, gross & net margins, ROCE, ROE.
Working capital dashboard: DSO, DIO, DPO, cash conversion cycle and rolling 12-month trends.
Profit bridge: waterfall showing drivers of YoY/period change (volume, price, cost, FX, one-offs).
Controls, segregation of duties & governance
Segregation of duties: separate roles for prepare, review and approve journals and adjustments; maintain read-only published reports.
Access control: restrict edit rights to calculation layers; protect input sheets but allow designated roles to edit.
Change log & policy control: document policy changes (depr lives, thresholds) with retro impact analysis.
Common pitfalls & defensive coding patterns
Pitfalls: hard-coded numbers in published FS, broken external links, forgotten query refresh, inconsistent mapping after CoA change, unreconciled control accounts.
Defensive patterns: IFERROR/IFNA on support sheets only, assertion checks that block publication if TB ≠ 0, central parameters, unit-test style checks for sample transactions.
Documentation, training & handover
Model Guide: short sheet with purpose, data sources, key assumptions, contact owners, refresh steps and known limitations.
Handover pack: list of critical schedules, login/refresh steps and runbook for month-end close.
Training: recorded demos for the close process plus a persistent FAQ and troubleshooting list.
Automation & tooling roadmap
Short-term: template hardening, mapping table completion, automated checks, Power Query ingestion.
Medium-term: Power Pivot/data model or small database back-end for multi-entity consolidation; automated TB imports; parameterised close macro to run checks.
Long-term: integrate GL export to BI/ETL pipeline, continuous reconciliations, dashboarding and workflow approvals.
Final checklist before publication — must-pass items
Must-pass: TB reconciled; AR/AP reconciled and aged; bank reconciled to GL; manual journals documented & approved; tax provision supported; FA schedule reconciled; disclosures drafted; control sheet shows checks passed or explained.
Sign-off: preparer, reviewer and C-level sign-off recorded and saved with the published file.
Examples & hands-on templates recommended
Templates to maintain: GL import template, mapping table, adjustment register, FA roll-forward, debt schedule, tax provision schedule, indirect cashflow template, central control & exceptions sheet, model guide.
Sample exercises: run a mock month-end with seeded errors to test controls; walkthrough a complex manual journal from trigger to posting to reporting.

Chapter 3 — Calculating and Projecting Various Financial Ratios
Overview & Purpose
• Why ratios matter: rapid diagnostics of liquidity, solvency, profitability, efficiency and valuation; early warning for covenant breaches; inputs to valuation, credit analysis and working-capital forecasting.
• Use cases: board / lender pack, management KPI dashboard, sensitivity testing, M&A pre-screen, covenant monitoring.
• Core principle: ratios are meaningful only relative to peers, trends and accounting-policy-consistent bases. Always document source line items and calculation conventions.
Required inputs and data-preparation rules
Source lines
• Income statement: Revenue (net of returns), COGS, Gross Profit, Operating Expenses, D&A, EBIT, Interest, EBT, Tax, Net Income.
• Balance sheet: Cash & equivalents, Trade receivables, Inventory, Trade payables, Other current assets/liabilities, Fixed assets (gross & accumulated depreciation), Total debt (short + long), Shareholders' equity.
Preparation rules
• Use audited or management-prepared trial balances where possible.
• Normalize one-offs (extraordinary gain/losses), present both adjusted and unadjusted ratios.
• Convert all currencies to reporting currency; note FX effects.
• Use averages for balance-sheet denominators: Average = (Opening + Closing) / 2 unless monthly rolling average required.
• Day base: choose 360 or 365 and keep it consistent across the workbook.
Conventions, rounding and presentation
• Show ratios to 1–2 decimal places (margins % to 1 decimal; days to 0 decimals).
• For reporting, separate raw calculation (hidden) and presentation (visible).
• Use named ranges or Tables for robust formulas; avoid hard-coded cell addresses for portability.
• Keep a “Definitions & Assumptions” sheet listing each ratio formula, source cells, day base and special adjustments.
Liquidity ratios — definitions, formulas, Excel patterns, interpretation
Current ratio
• Formula: Current Assets / Current Liabilities
• Excel pattern (structured refs): =SUM(TableBS[Current Assets]) / SUM(TableBS[Current Liabilities]) or =B4/B10
• Interpretation: >1 usually comfortable; industry norms vary. Watch seasonal swings.
Quick ratio
• Formula: (Current Assets − Inventory) / Current Liabilities
• Excel: =(B4-B7)/B10
Cash ratio
• Formula: (Cash + Cash Equivalents) / Current Liabilities
• Use in stress tests and covenant modelling.
Checks & caveats
• Exclude restricted cash from numerator unless available for debt service. Document any reclassifications.
Solvency ratios — definitions, formulas, Excel patterns, interpretation
Debt to Equity
• Formula: Total Debt / Shareholders' Equity
• Excel: =(ShortTermDebt+LongTermDebt)/Equity
Debt to Assets
• Formula: Total Debt / Total Assets
Debt to Capital
• Formula: Total Debt / (Total Debt + Equity)
Interest Coverage Ratio (ICR)
• Formula: EBIT / Interest Expense
• Excel: =IF(Interest=0,NA(),EBIT/Interest) — show NA or large value if interest = 0.
Debt Service Coverage Ratio (DSCR)
• Formula: Operating Cash Flow / (Principal Repayments + Interest)
• Use cash-flow statement lines, not P&L interest accrual only.
Interpretation
• Lenders focus on ICR/DSCR minimum thresholds; show historical cushion and projected path.
Profitability ratios — definitions, calculation notes, Excel tips
Gross profit margin
• Formula: Gross Profit / Revenue
Operating profit margin (EBIT margin)
• Formula: EBIT / Revenue
Pre-tax profit margin
• Formula: EBT / Revenue
Net profit margin
• Formula: Net Income / Revenue
Return on Assets (ROA)
• Formula: Net Income / Average Total Assets
Return on Capital Employed (ROCE)
• Formula: EBIT / (Total Assets − Current Liabilities) — document chosen denominator.
Return on Equity (ROE)
• Formula: Net Income attributable to equity / Average Equity
Excel tips
• Use AVERAGE() on opening/closing balances or use rolling averages for monthly forecasting.
Turnover / efficiency ratios — definitions, formulas, days conversions, Excel implementation
Inventory turnover
• Formula: COGS / Average Inventory — or Net Sales / Average Inventory depending on convention; chapter uses Net Sales for high-turn consumer examples.
Receivables turnover (AR)
• Formula: Net Credit Sales / Average Receivables (if credit sales known) or Net Sales / Average Receivables.
Payables turnover (AP)
• Formula: Purchases / Average Payables or Net Sales / Average Payables (if purchases not available).
Fixed assets turnover
• Formula: Net Sales / Average Net Fixed Assets.
Convert to days
• DSO = DayBase / Receivables Turnover OR = Average Receivables / (Credit Sales / DayBase).
• DIO = DayBase / Inventory Turnover OR = Average Inventory / (COGS / DayBase).
• DPO = DayBase / Payables Turnover OR = Average Payables / (Purchases / DayBase).
Excel patterns
• Ensure denominators use AVERAGE(open,close) or rolling average; for monthly models use AVERAGE(range) over N months.
Cash Conversion Cycle — formula, use-cases, modelling guidance
Formula
• CCC = DSO + DIO − DPO.
Use
• Measures days cash tied up in working capital; drives short-term funding need. Integrate CCC into cash-flow and debt schedules.
Forecasting
• Forecast DSO, DIO, DPO directly (driver approach) or forecast receivables/inventory/payables from sales and ratios. Link changes in working capital to cash flow.
Valuation ratios — definitions, formula detail, modelling notes
EPS (basic & diluted)
• Basic EPS = (Net Income − Preferred Dividends) / Weighted Average Shares Outstanding.
• Diluted EPS includes dilutive securities per IFRS/US GAAP rules — model separately and disclose assumptions.
Price multiples
• P/E = Market Price / EPS.
• P/B = Market Price / Book Value per Share.
• P/S = Market Cap / Revenue.
Enterprise Value and enterprise multiples
• EV = Market Cap + Total Debt − Cash & Equivalents + Minority Interest + Preferred Stock (if any).
• EV/EBITDA, EV/Sales are preferred when capital structure differs. Use LTM (last twelve months) or forward estimates consistently.
Interpretation & caveats
• Multiples are market-driven; use peer medians and justify discrepancies (growth, margin, capital intensity).
Forecasting methodology — step-by-step process & formulas
1. Define horizon and cadence
• Monthly for short-term cash needs, quarterly for operational planning, annual for strategic/valuation.
2. Forecast revenue
• Choose driver (volume × price, or top-down growth rates). Document seasonality patterns and base-year adjustments.
3. Forecast margins
• Apply margin drivers: COGS% of revenue, OpEx% of revenue, scenario-based changes (efficiency, inflation pass-through).
4. Forecast working capital using days or % drivers
• Preferred approach: forecast DSO/DIO/DPO as driver cells; compute receivables/inventory/payables from sales/COGS.
• Example formula: Forecast_AR = Forecast_Revenue / DayBase * Forecast_DSO (rearranged if needed).
5. Forecast capex & depreciation
• Capex schedule by year; compute closing fixed assets = opening + capex − depreciation. Use depreciation policy consistent with accounting policy (SL or DB).
6. Debt & interest schedule
• Project borrowings, principal repayments, interest calc on opening debt or average debt per covenant; map interest expense into P&L and cash interest into cash flow.
7. Calculate ratios on forecasted statements
• Use average balances where needed; present base/best/worst cases.
Scenario & sensitivity analysis — recommended practice
• Build a scenario switch (Best / Base / Worst) controlling key drivers: revenue growth, gross margin, DSO/DIO/DPO, capex, and financing cost.
• Use one-way sensitivity tables and two-variable data tables for key ratios (e.g., IRR vs revenue growth and margin).
• Produce tornado charts for top drivers of ratio variance.
Covenant modelling & alerts
• Implement covenant formulas exactly as in loan docs (e.g., minimum ICR of 3.0x, maximum Net Debt/EBITDA of 3.5x).
• Create “covenant flag” boolean cells that turn red when breached and a summary table showing historical cushion and forecast breach month.
• Model minimum cash triggers and automatic drawing mechanics if covenants breach.
Presentation, dashboards and visual checks
• Visuals to include: ratio trend lines (3–5 years), heatmap of ratios vs peer quartiles, CCC trend, leverage waterfall, covenant dashboard with red/amber/green flags.
• Use pivot charts & slicers for interactive reporting; expose scenario selector on dashboard.
• Use conditional formatting to highlight outliers and breaches.
Excel implementation details, formulas & robustness tips
• Use Tables (Ctrl+T) and structured references for source data.
• Name key inputs (e.g., DayBase, DSO_Target) and lock assumption cells on the assumptions sheet.
• Use IFERROR/IF defensively: =IF(denominator=0,NA(),numerator/denominator) to avoid spurious zeros.
• Use ROUND only for presentation; keep raw values in calculations.
• Use INDEX/MATCH or XLOOKUP for dynamic lookups; avoid VLOOKUP with static column indexes.
Reconciliation, validation & audit checks
• Reconcile totals to source (trial balance) quarterly and annually.
• Validation checks to include: Assets = Liabilities + Equity, Cash flow reconciliation (Net income → Net cash), Movement check for working-capital lines (ΔReceivables = ForecastReceivables − PriorReceivables), Sum of quarterly equals year-to-date totals.
• Build a “validation panel” listing PASS/FAIL for each check; require green before distribution.
Common pitfalls, causes and corrective actions
• Using year-end balances instead of averages → distort turnover; fix: use averages.
• Mixing cash vs accrual sales for DSO → inconsistent days; fix: choose credit sales or adjust formula.
• Ignoring seasonality → cash shortages in peak months; fix: monthly modelling and seasonal indices.
• Including non-recurring items in margins → misleading profitability; fix: show adjusted ratios and disclose items.
Worked numeric walkthroughs (detailed calculations using chapter example)
Given: Current assets turnover = 196%, Debtors turnover = 400%, Creditors turnover = 308%, Inventory turnover = 385% (chapter examples). DayBase = 365.
Compute DSO
• AR Turnover = 400% → Turnover ratio = 4.00 (times per year).
• DSO = 365 / 4.00 = 91.25 → 91 days (round to nearest day).
Compute DPO
• AP Turnover = 3.08 → DPO = 365 / 3.08 = 118.51 → 119 days.
Compute DIO
• Inventory Turnover = 3.85 → DIO = 365 / 3.85 = 94.81 → 95 days.
CCC
• CCC = 91 + 95 − 119 = 67 days (chapter example).
Interpretation
• A 67-day CCC means cash is tied for ~9.5 weeks; management must fund that gap via working-capital, supplier terms or inventory optimisation.
Deliverable checklist before handing off model/report
• Assumptions sheet present and locked.
• All input cells named and highlighted; formulas protected but inputs editable.
• Validation panel all green or exceptions documented.
• Scenario selector works and updates all outputs.
• Pivot refresh instructions and macros signed or documented.
• Version control: saved as ModelName_vX.Y_YYYYMMDD.xlsx and backup copy stored.
Audit trail and documentation requirements
• Keep a change log tab: date, user, change summary, cells affected.
• For each ratio, document calculation, source cells, and known limitations.
• Attach source extracts (trial balance, reconciliations) as supporting pages or PDFs for external reviewers.
Advanced notes — adjustments, industry specifics, and extensions
• Banking/financial firms: use regulatory capital measures and different denominator rules (e.g., risk-weighted assets).
• Oil & gas / extractive: use unit-of-production depreciation for asset-intensive ratios and forecast commodity price sensitivity separately.
• Seasonality & cyclical firms: use rolling twelve-month ratios and seasonal indices; consider median-of-months rather than averages for volatile items.
Final recommendations & governance
• Standardise ratio definitions in a corporate glossary to ensure consistent reporting across teams.
• Review ratio inputs monthly and reconcile to general ledger each quarter.

Chapter 4 — Modeling Working Capital
Introduction
Working capital is the primary fuel for a business; mismanagement can cause liquidity problems and even insolvency. Chapter explains working capital management, importance, types, components, calculation of debtors’ collection period, creditors’ payment period, inventory conversion period, and the cash conversion cycle (operating cycle). The chapter also covers sources of working capital finance and bank assessments for working-capital limits.

Objectives
Explain why working capital matters; teach calculation of debtors’ collection period, creditors’ payment period, inventory conversion period; explain cash conversion cycle, its drivers, how to manage working capital efficiently; describe methods banks use to assess working-capital requirements.
What is Working Capital Management
Definition
Working capital = Current Assets − Current Liabilities. It is money used to cover day-to-day operating expenses, short-term debts/obligations and inventory expenses. The current ratio (current assets / current liabilities) is often called the working-capital ratio — a current ratio < 1 is an alarm sign.
Goal
Minimise required working capital while ensuring smooth operations (shortest possible operating / cash conversion cycle). Good management balances inventory, receivables and payables to maximize liquidity and returns.
Importance of Working Capital
● Higher Return on Capital Employed — achieve higher returns with lower capital tied up.
● Improvement in Solvency & Credit Profile — meeting short-term dues improves creditworthiness.
● Better Liquidity — ability to meet obligations during crises (e.g., pandemic).
● Increase in Profit & Business Expansion — efficient receivables/payables management raises profitability and supports growth.
● Edge over Competitors — better working-capital management enables competitive pricing and trade terms.
● Additional benefits: better financing terms from banks, uninterrupted operations, improved goodwill.
Types of Working Capital
● Gross Working Capital — total current assets (does not consider liabilities).
● Net Working Capital — current assets − current liabilities (core concept).
● Positive Working Capital — current assets > current liabilities (surplus).
● Negative Working Capital — current liabilities > current assets (deficit).
● Permanent / Fixed Working Capital — minimum floor of working capital that does not change short-term (predictable).
● Temporary / Variable Working Capital — fluctuates seasonally or with special orders (harder to predict).
Components of Working Capital
Current Assets (realised within 1 year): cash & cash equivalents; bank deposits; marketable securities; short-term investments; accounts receivable (trade receivables); inventory; prepaid expenses; short-term loans & advances.
Current Liabilities (paid within 1 year): accounts payable (trade payables); short-term loans; cash credit and overdrafts; advances from customers; outstanding expenses; current portion of term loans / instalments.
Operating Cycle / Cash Conversion Cycle (Overview)
Defines time taken for cash → raw material → WIP → finished goods → sales → cash. This operating cycle (cash conversion cycle, CCC) shows how long cash is tied up before returning. Shorter CCC is better. Example manufacturing flow: cash buys raw material (30 days) → raw → WIP (10 days) → WIP→finished (20 days) → finished to sale (10 days) → sale to cash (20 days) → total 90 days in example.
Key turnover ratios (basis for periods)
Turnover ratios are normally computed as Net Sales ÷ Average of opening & closing balance (average preferred).
● Current Assets Turnover Ratio = Net Sales / Average Current Assets — measures sales generated per unit of current assets.
● Debtors Turnover Ratio (Accounts Receivable Turnover) = Net Sales / Average Debtors — measures speed of collections.
● Creditors Turnover Ratio (Accounts Payable Turnover) = Net Sales / Average Creditors — measures how fast company pays suppliers.
● Inventory Turnover Ratio = Net Sales / Average Inventory — measures how well inventory is utilised.
Debtors / Trade Receivable
Definition
Trade receivables = value of goods/services sold on credit (invoices issued).
Debtors Collection Period / Days Sales Outstanding (DSO)
Formula (common): DSO = 365 / Debtors Turnover Ratio (some analysts use 360). It indicates average days to collect receivables. A high DSO = slower collections; low DSO = faster collections.
Trade Payable / Creditors
Definition
Trade payables = value of goods/services bought on credit (invoices received).
Creditors Payment Period / Days Payable Outstanding (DPO)
Formula (common): DPO = 365 / Creditors Turnover Ratio (360 sometimes used). It indicates average days to pay suppliers. Higher DPO indicates longer supplier credit.
Inventory
Definition
Inventory = stocks of raw materials, WIP, finished goods.
Inventory Conversion Period / Days Inventory Outstanding (DIO)
Formula (common): DIO = 365 / Inventory Turnover Ratio (360 sometimes used). It indicates average days inventory is held before sale. Lower DIO = faster inventory turnover.
Cash Conversion Cycle (CCC) — formula & interpretation
Formula
CCC = Debtors Collection Period (DSO) + Inventory Conversion Period (DIO) − Creditors Payment Period (DPO).
Interpretation
CCC measures how many days cash is tied up in the operating cycle. Compare CCC to peers and historical trend. Shorter CCC means faster cash recovery; longer CCC means more working capital tied up.
Cash Conversion Cycle — Worked example (step-by-step)
Step 1: Compute averages (Average of opening & closing balances) for debtors, creditors, inventory.
• Example averages: Debtors = 25, Creditors = 32.5, Inventory = 26.
Step 2: Compute turnover ratios (Net Sales ÷ average):
• Example results: Debtors turnover = 4, Creditors turnover = 3.08, Inventory turnover = 3.85.
Step 3: Convert turnover into days (periods):
• DSO = 365 / 4 = 91 days.
• DPO = 365 / 3.08 = 119 days.
• DIO = 365 / 3.85 = 95 days.
Step 4: Compute CCC = DSO + DIO − DPO = 91 + 95 − 119 = 67 days.
Conclusion from example — company takes 67 days to complete the operating cycle (cash → cash).
Determination of Working Capital (factors)
Key determinants that influence required working capital:
● Nature & Size of Business — service firms need less working capital than trading/manufacturing; manufacturing needs most.
● Position in Business Cycle / Seasonality — festive seasons or large orders increase WC needs.
● Manufacturing Process — longer production cycles → higher WC.
● Market Conditions — high competition may require bigger inventory to sell at lower prices.
● Growth & Expansion — more WC required for expansion.
● Credit Policy — tight customer credit policy reduces WC; tight supplier credit increases WC need.
● Dividend Policy — dividends reduce cash (increase WC requirement); retained earnings increase internal WC source.
● Tax Level — tax rates and provisions affect WC.
● Other Determinants — coordination issues between departments, transport/infrastructure delays, import-export policies.
Sources of Working Capital Finance
Financing without loans (internal / trade)
● Trade Credit (supplier credit).
● Advances from customers for orders.
● Accrued expenses (defer payment of outstanding expenses).
● Provisions (e.g., bonus, taxes) used until payable.
Short-Term Loans & Credit Facilities
● Cash Credit (secured by inventory & receivables; interest on daily balance).
● Overdraft facility (withdraw beyond account balance).
● Short-term loans / working-capital loans from banks.
● Bill discounting / purchase of bills (banks advance against receivables).
● Pre-shipment & post-shipment finance for exporters.
Non-Fund Based Facilities
● Letter of Credit (LC), Bank Guarantee (BG), Buyers Credit — support trade without immediate cash movements.
Long-Term Financing
● Equity (VC, public equity).
● Term loans, loans against property or securities.
Working Capital Assessment (bank practice & methods)
Banks and financiers usually assess WC requirement using several standard methods: MPBF (Maximum Permissible Bank Finance), Operating Cycle Method, Drawing Power Method, Turnover Method, Cash Budget Method. These methods produce the sanctioned working-capital limit and conditions.
Maximum Permissible Bank Finance (MPBF)
Used for companies requiring up to certain thresholds; banks usually finance up to 75% of required WC with remaining from long-term sources.
Method I
MPBF = 75% of (Current Assets − Current Liabilities other than bank borrowings).
Ideal Current Ratio = 1.0 : 1.
Method II
MPBF = (75% of Current Assets) − (Current Liabilities other than bank borrowings).
Ideal Current Ratio = 1.33 : 1.
Example
Method I MPBF = 1,200 (units) and Method II MPBF = 1,100 (units) in the chapter example — shows Method I is more liberal and provides higher sanction.
Operating Cycle Method (bank)
Banks compute number of operating cycles in a year and divide annual operating expenses by cycles to estimate WC requirement.
Formula
Working Capital Requirement = Annual Operating Expenses / No. of Operating Cycles.
Example
If operating cycle = 67 days → cycles per year = 365/67 = 5.45. If annual operating expense = 120 lakh → WC requirement = 120 / 5.45 ≈ 22.02 lakh.
Drawing Power Method (bank)
Concept
Drawing Power (DP) is the monthly/periodic amount a borrower can withdraw under cash-credit limit based on collateral (inventory, receivables) and applicable margins. DP is reviewed monthly and cannot exceed the sanctioned limit.
Formula (generalised)
Drawing Power = (Stock − Creditors − WC Margin) + (Book Debts − Margin) where margin may be 25%–40% depending on bank policy and asset class. Debtors older than 90 days may be excluded. Month 1 values determine Month 2 DP, and so on. DP ≤ Sanction Limit.
Example
Month 2 DP = 60 lakh; Month 3 DP = 109.5 lakh but Sanction Limit = 100 lakh → borrower cannot withdraw >100 lakh despite DP being 109.5 lakh; practical withdrawal limited to sanction.
Turnover Method (for MSMEs)
Used for small businesses (e.g., turnover ≤ ₹5 crore in example). WC requirement taken as a percentage of projected turnover.
Rule in example
Working Capital gap = 25% of Projected Turnover (of which 5% is margin / net WC and 20% is bank financing = MPBF).
Example
Projected turnover = ₹1 crore → WC gap = 25% = ₹25 lakh; Margin (net WC) = 5% = ₹5 lakh; MPBF = 20% = ₹20 lakh. Simple and useful for small enterprises.
Cash Budget Method (seasonal / cyclical businesses)
Applicable where cash inflows/outflows fluctuate seasonally. Bank finances peak deficits based on projected cash flow. Company submits last year actual cash flow + 1-year projected cash flow; MPBF based on peak deficit months.
Example
Projected monthly cash flow shows maximum deficit ₹70 lakh in October → Working Capital Limit = Peak Deficit = ₹70 lakh.
Working Capital Assessment — Worked examples & steps
Stepwise approach used in chapter:
1. Compute averages of opening & closing balances of debtors, creditors, inventory.
2. Compute turnover ratios using Net Sales and averages.
3. Compute collection/payment/inventory periods (DSO, DPO, DIO).
4. Compute Cash Conversion Cycle (CCC = DSO + DIO − DPO).
5. Use bank methods (MPBF / Operating Cycle / Drawing Power / Turnover / Cash Budget) to arrive at bank finance limits and sanction terms.
Practical tips & caveats
● Use average of opening and closing balances (preferred) rather than just closing balances for more reliable turnover ratios.
● Some analysts use 360 days for period conversions; chapter uses 365 (state which you use consistently).
● Exclude very old receivables (e.g., >90 days) when calculating drawing power — banks often ignore them.
● Maintain coordination across purchase, production, sales, and finance teams to reduce WC needs.
● Always compare CCC and other metrics against industry peers and historical trends.
Conclusion
Working capital management is essential for business survival and growth. Proper calculation of turnover ratios (debtors, creditors, inventory), resulting periods (DSO, DPO, DIO), and the cash conversion cycle helps determine WC needs. Multiple bank assessment methods (MPBF, operating-cycle, drawing-power, turnover, cash-budget) exist; choose the appropriate method based on company scale and seasonality. Efficient WC management improves liquidity, solvency, profitability, and competitive position.

Chapter 5 — Preparing Business Valuation Modeling
Overview & Purpose
Explains the core valuation concepts and practical modelling needed to evaluate projects and entire businesses. Emphasis on: time value of money (TVM), cash-flow based valuation (DCF/FCF), capital budgeting metrics (NPV, IRR, PI, payback), accounting metrics (ARR), cost of capital (CAPM, WACC), terminal value methods, Excel best practices for each calculation, scenario & sensitivity analysis (Data Tables, Goal Seek), and worked Excel examples with numeric inputs and formulas. The chapter teaches both the theory and the exact Excel steps required to build auditable valuation models.
Time Value of Money (TVM)
Concept
• Money now ≠ money later — future amounts must be discounted because of opportunity cost and inflation; TVM underlies FV/PV, discounting for valuation and capital budgeting.
Future Value (FV)
• Formula: FV = PV * (1 + i/n)^(n*t) where PV = present value, i = nominal annual rate, n = compounding periods per year, t = years.
• Worked numeric example: PV = ₹100,000; i = 8% p.a.; n = 1; t = 10 → FV = 100000*(1+0.08)^10 = 215,892.50.
• Excel: =FV(rate, nper, pmt, [pv], [type]) — example =FV(B3/B4, B4*B5, 0, -B2); use negative PV when modeling cash outflow; divide rate by n and multiply periods by n for non-annual compounding.
Present Value (PV)
• Formula: PV = FV / (1 + i/n)^(n*t).
• Worked numeric example: FV = ₹200,000; i = 8% p.a.; n = 1; t = 10 → PV ≈ ₹92,638.70.
• Excel: =PV(rate, nper, pmt, [fv], [type]) — example =PV(B3/B4, B4*B5, 0, -B2).
Compounding frequency
• For quarterly/ monthly compounding convert rate and nper: rate_per_period = annual_rate / periods_per_year; nper = years * periods_per_year — ensure Excel formulas reflect this.
Net Present Value (NPV) and Internal Rate of Return (IRR)
NPV concept & interpretation
• NPV = Σ (CF_t / (1+r)^t) − InitialInvestment where r = discount rate; NPV in currency units measures value added.
• Decision rule: NPV > 0 accept; NPV < 0 reject. Use WACC (or project-specific discount rate) consistent with cash-flow type (levered/unlevered).
Worked NPV numeric example (as used in chapter)
• Initial cost = ₹1,00,00,000 (₹1 crore). Cash inflows in lakhs: Yr1=20, Yr2=30, Yr3=30, Yr4=35, Yr5=25, Yr6=10 + scrap 15 → Yr6 total 25. Discount rate = 10% → NPV ≈ ₹19.06 lakh (positive → accept).
Excel NPV function caveat & correct pattern
• Excel =NPV(rate, values...) discounts the values provided starting at period 1; do NOT include the initial investment in the values argument. Correct pattern: =NPV(discount_rate, Year1:YearN) + InitialCashFlow (InitialCashFlow usually negative). Example used: =NPV(C2, D7:I7) + C7. Be explicit with signs.
IRR concept & interpretation
• IRR is the rate r that makes NPV = 0. Interpreted as project's compounded annual return. Accept if IRR > hurdle rate/required return.
IRR worked example & Excel methods
• Same cashflow → IRR ≈ 16.09%.
• Methods: (a) =IRR(range_of_cashflows,[guess]) including initial outflow and subsequent flows; (b) Goal Seek: set NPV cell = 0 by changing discount rate cell (demonstrated step-by-step in chapter).
IRR warnings & multi-IRR cases
• Non-conventional cash flows (multiple sign changes) can produce multiple IRRs; IRR may be misleading for mutually exclusive projects with different scales — prefer NPV or Modified IRR (MIRR) for comparison.
Profitability Index (PI)
Definition and usage
• PI = PV of future inflows / PV of outflows = (NPV + InitialInvestment) / InitialInvestment. A PI > 1 means positive NPV per unit of investment; useful when capital is rationed.
Worked example
• Using earlier cashflows: PI ≈ 1.19 (interpreted as project delivering 1.19× PV of outflows).
Payback Period and Discounted Payback
Payback Period
• Time to recover initial investment from cumulative nominal cash inflows. Fast and intuitive but ignores TVM and post-payback cashflows.
• Example: nominal payback = 3.43 years (compute cumulative inflows year-by-year; if recovery inside a year, partial year = remaining / cashflow in recovery year).
Discounted Payback
• Same idea but using discounted cashflows (PV). Better because it considers TVM; example: discounted payback ≈ 4.32 years at 10% discount rate.
Accounting Rate of Return (ARR)
Definition & calculation conventions
• ARR = Average annual accounting profit / Average investment. Average investment often = (Initial cost + Salvage)/2 under Straight-Line Depreciation assumptions.
Worked example and rule
• Investment ₹6,00,000, salvage ₹1,50,000, life 6 years, given profits → ARR ≈ 27.11% which is less than cost of capital 28% → reject under ARR rule. Notes: ARR uses accounting profits (non-cash D&A included) and ignores TVM; useful as an accounting sanity check, not primary decision metric.
Capital Asset Pricing Model (CAPM) — cost of equity
CAPM formula
• Cost of equity = Rf + β × (Rm − Rf) where Rf = risk-free rate, β = asset/stock beta, Rm = expected market return.
Worked example
• Rf = 6%, Rm = 10%, β = 1.2 → Cost of equity = 6% + 1.2*(10%−6%) = 10.8%.
Practical notes
• Beta sourcing: financial data providers (use levered/unlevered beta carefully).
• Use CAPM for equity investors’ required return; combine with debt cost to compute WACC.
Weighted Average Cost of Capital (WACC)
Formula
• WACC = (E/C)×Re + (D/C)×Rd×(1−Tc) where E = market value of equity, D = market value of debt, C = E + D, Re = cost of equity, Rd = cost of debt, Tc = tax rate.
Worked numeric example (three capital structure options)
• Need ₹150 lakh total capital.
• Option 1: 100% debt @ 12% → WACC ≈ 12% (pre-tax; post-tax effect often reduces effective cost if debt tax-deductible).
• Option 2: 50% debt @ 12% + 50% equity @ 20% → WACC ≈ 16% (illustrative numbers).
• Option 3: blended instruments (loans, bonds, equity, preferred) → compute each component weight × cost and sum; example yielded WACC ≈ 13.31% applying (1−T) to debt components.
Decision rule
• Discount project cashflows at WACC (for unlevered FCF). Accept if NPV (discounted at WACC) > 0 and IRR > WACC. Be careful: use WACC for unlevered cash flows; discounting levered cash flows requires cost of equity.
Business Valuation Approaches
Overview
• Three primary approaches: Comparable Company Analysis (market multiples), Precedent Transactions (M&A multiples), Discounted Cash Flow (intrinsic valuation). DCF is emphasised for project and going-concern valuation. Use all three for triangulation where possible.
Comparable Company Analysis
• Select peer universe, compute relevant multiples (EV/EBITDA, EV/Sales, P/E, P/B), adjust for accounting differences, apply peer median/percentile to target metrics to infer EV/Equity value.
Precedent Transactions
• Use transaction multiples paid in real deals (control premium applies), adjust for timing, deal structure and synergies; useful for M&A pricing but fewer comparable events may exist.
Discounted Cash Flow (DCF)
• Focus on detailed projection of FCF over explicit forecast horizon, then compute terminal value (exit multiple or Gordon growth) and discount using appropriate discount rate (WACC for unlevered FCF). DCF produces Enterprise Value (EV) which converts to Equity Value by subtracting Net Debt.
Free Cash Flow (FCF) — definitions & calculation methods
Which FCF to use?
• FCFF (Free Cash Flow to Firm / unlevered FCF): cash available to all capital providers (discount with WACC).
• FCFE (Free Cash Flow to Equity / levered FCF): cash available to equity holders after debt servicing (discount with cost of equity). Choose one and be consistent.
Three practical calculation methods
1. FCF = Operating Cash Flow − Capital Expenditure (simple, uses cash flow statement).
2. FCF = Sales − (Operating Costs + Taxes) − Investment in Operating Capital (useful when building from gross margins and WC assumptions).
3. FCF = EBIT × (1 − Tax Rate) + Depreciation − CapEx − ΔWorking Capital (builds from P&L items; common in DCF).
Working capital & depreciation
• ΔWorking Capital = change in (Trade Receivables + Inventory + Other current assets − Trade Payables − Other current liabilities).
• Depreciation is non-cash and added back; CapEx is cash outflow reducing FCF. Be explicit about timings (year-begin vs year-end conventions) and consistency with revenue timing.
Discounted Cash Flow (DCF) — step-by-step modelling
Step-by-step process
• 1. Build forecast income statement (revenue drivers, margins, growth rates).
• 2. Forecast operating items: COGS, OpEx, EBITDA, D&A (use capex/schedule), EBIT.
• 3. Forecast taxes: use EBT/EBIT and applicable tax rate; consider tax-loss carryforwards, special tax regimes if relevant.
• 4. Forecast CapEx and compute D&A schedule (link capex to fixed asset roll-forward and depreciation policy).
• 5. Forecast working capital changes by modelling receivables, inventory and payables days (DSO, DIO, DPO) or absolute balances.
• 6. Compute unlevered FCF: FCFF = EBIT × (1−Tc) + Depreciation − CapEx − ΔWorking Capital (or use Operating Cash Flow − CapEx).
• 7. Select discount rate (WACC) and discount each FCFF to present: PV_CF_t = FCFF_t / (1 + WACC)^t.
• 8. Compute terminal value (exit multiple or perpetuity growth) and discount it back to present.
• 9. Sum PV of forecasted FCFFs and PV of terminal value → Enterprise Value (EV).
• 10. Adjust for non-operating items and net debt: Equity Value = EV − Net Debt − Minority Interests + Cash-like items.
• 11. Divide by diluted shares outstanding → implied share price; perform sanity checks vs comparables.
Terminal value methods & formulas
Gordon Growth (Perpetuity) Method
• TV = FCF_last × (1 + g) / (r − g) where g = perpetual growth rate (conservative — often GDP/long-term inflation rate), r = discount rate (WACC).
Exit Multiple Method
• TV = EBITDA_last × ExitMultiple where ExitMultiple derived from comparable company or precedent transaction medians (EV/EBITDA).
Present value of terminal value
• PV_Terminal = TV / (1 + r)^n where n = final forecast year. Document chosen approach and run sensitivity across g and multiples.
Worked DCF example (numbers & decision)
Inputs
• WACC = 15%; Initial investment = ₹12 lakh; Yearly CFs: Yr1=1.5L, Yr2=2L, Yr3=4L, Yr4=5L, Yr5=7L.
Results & interpretation
• Discounted cashflow sum produced net adjusted DCF ≈ −0.79 (negative) at initial investment ₹12L → reject at that price. If initial investment reduced to ₹9L, net adjusted DCF ≈ 2.21 (positive) → acceptable. Emphasize sensitivity to initial price and discount rate.
Terminal & residual valuation adjustments
• Include salvage value, working capital recoveries, and final-year adjustments (e.g., last-year CapEx required to maintain steady state) in final year cashflow used for terminal calculation. Always discount terminal value separately.
Tax treatment, depreciation & amortisation in DCF
• Use EBIT × (1 − TaxRate) for operating-profit based FCF computations (unlevered).
• Depreciation is added back to taxable profit because it's non-cash, but CapEx is subtracted in the FCF line (cash out).
• For tax shield considerations (if valuing levered cash flows), consider interest tax shield impact, but prefer unlevered valuation and adjust capital structure separately.
Unlevered vs Levered cash flows & discount rates
• Unlevered FCF (FCFF) discounted at WACC → gives Enterprise Value (EV).
• Levered FCF (FCFE) discounted at cost of equity → gives Equity Value directly. Pick one consistent approach and avoid mixing.
Free Cash Flow (FCF) example calculations
• Method example: FCF = EBIT*(1−Tc) + Depreciation − CapEx − ΔWC — compute each term from forecast schedules.
• Example in chapter produced positive FCF ₹12,50,000 indicating ability to invest or pay dividends.
PMT, amortisation & Data Table (Excel scenario grids)
PMT / EMI calculation
• Excel PMT: =PMT(rate, nper, pv, [fv], [type]). Example monthly EMI = =ROUND(PMT(8%/12, 5*12, -LoanAmount),2) → EMI ₹4,866.33 for 5 years @ 8% p.a. Total paid ~ ₹291,980.08.
Loan schedule construction
• Use IPMT and PPMT to split each payment: =IPMT(rate, per, nper, pv) for interest portion and =PPMT(rate, per, nper, pv) for principal portion. Build period-by-period table with OpeningBalance → Payment → Interest → Principal → ClosingBalance.
Data Table for sensitivity
• Steps: lay rates across top row and periods/years down left column; place base PMT cell at top-left corner of table; select entire table range → Data → What-If Analysis → Data Table → set Row input cell and Column input cell to the appropriate input cells (rate cell and nper cell). Data Table returns matrix of outputs (EMIs, NPV or any formula) across combinations.
Goal Seek & IRR via Goal Seek
• Goal Seek process example: set NPV cell = 0 by changing DiscountRate cell. Excel path: Data → What-If Analysis → Goal Seek → Set cell = (NPV cell), To value = 0, By changing cell = (DiscountRate cell) → OK → solution yields IRR. Useful when IRR() returns ambiguous value or when you want to force NPV to exact zero with custom NPV structure.
NPV/IRR pitfalls & Excel best-practices
• Always check sign conventions in PV/FV/PMT functions (cash outflows are negative).
• Do not place initial investment inside the NPV() argument; add separately (or use XNPV if dates are irregular).
• For irregular timing, use XNPV and XIRR with explicit date ranges: =XNPV(rate, values, dates) and =XIRR(values, dates).
• Use MIRR (modified IRR) when you want to specify distinct reinvestment and finance rates, avoiding multiple-IRR problems: =MIRR(values, finance_rate, reinvest_rate).
• Break complex formulas into small auditable steps and leave intermediate schedules visible for checking (revenues, capex, WC, taxes, FCF).
Sensitivity & scenario analysis
Scenario analysis
• Create Base / Best / Worst scenarios by changing drivers (revenue growth, margins, capex intensity, working capital days, WACC). Use a scenario selector (drop-down) to switch inputs and recalculate outputs (NPV, IRR, EV, implied share price).
Sensitivity analysis
• 1-way sensitivity: vary one driver (e.g., WACC, terminal multiple, growth g) and record NPV/IRR outcomes; present in a table or chart (tornado chart).
• 2-way sensitivity: use a two-variable Data Table for NPV vs (WACC × TerminalMultiple) matrix.
Tornado charts & presentation
• Rank drivers by impact on NPV and plot bars to show which assumptions drive valuation. Always label axes and present ranges.
Terminal value sensitivity & conservative checks
• Test multiple terminal approaches (exit multiple band and perpetuity growth rates) and show valuation ranges. Use conservative perpetual growth (roughly long-run GDP + inflation or lower) and justify chosen exit multiples by peer median/percentile analysis.
Consolidation to enterprise & equity value
• After DCF obtains EV: adjust for non-operating assets/liabilities: subtract net debt (Debt − Cash), subtract minority interests, add investments or surplus cash → Equity Value. Divide by diluted shares outstanding for implied per-share value. Reconcile with comparable valuations.
Worked Excel patterns & formula snippets
• NPV pattern: =NPV(discount_rate, range_of_year1_to_n) + InitialInvestment (watch signs).
• IRR: =IRR(range_of_cashflows) or =XIRR(values, dates) for irregular timings.
• PV of terminal: =TerminalValue / (1 + discount_rate)^n or include terminal as final cash flow and apply NPV accordingly.
• PMT: =PMT(rate, nper, pv); IPMT/PPMT for schedule.
• FCFF from EBIT: =EBIT*(1-TaxRate) + Depreciation - Capex - ChangeInWC.
• WACC: compute component weights from market values (not book values) where possible.
Model auditing, sanity checks & reconciliation
• Reconcile model totals with source data and high-level sanity numbers (e.g., last 12 months revenue, EBITDA margins vs peer averages).
• Cross-check: sum of discounted FCFs + PV terminal = EV; EV − net debt = equity value; equity value / diluted shares = implied share price. Check that implied metrics (P/E, EV/EBITDA) fall into plausible ranges relative to peers.
• Use trace precedents/dependents, and show calculation steps on separate audit sheet for reviewers.
Worked examples & exercises recommended
• Recreate PV/FV examples and validate Excel FV/PV outputs.
• Build the example NPV/IRR timeline with scrap value and confirm IRR via both IRR() and Goal Seek.
• Build a full DCF model: forecast P&L → CapEx schedule → D&A schedule → working capital days → compute FCFF → discount at WACC → terminal value by both methods → EV → equity value → implied share price; then run 1-way and 2-way sensitivity tables on WACC and terminal multiple.
Presentation & communication of valuation
• Show base-case valuation plus sensitivity ranges (low/high).
• Provide clear assumptions page with drivers, tax rate, capex intensity, working capital days, depreciation policy, terminal assumptions and chosen multiples.
• Present reconciliation to market comparables and explain discrepancies (growth premium, margin differences, accounting adjustments).
Key takeaways and rules of thumb
• Always discount future cash flows — TVM is fundamental.
• Use unlevered FCF + WACC for firm valuation; FCFE + cost of equity for equity-only valuation.
• NPV is primary decision metric; IRR is complementary (watch for multiple IRRs).
• Terminal value typically dominates DCF — stress-test terminal assumptions carefully.
• Document every assumption; keep forecasts auditable and build modular schedules (revenue drivers, capex, WC, debt).
• Use Data Tables and Goal Seek for sensitivity and to test robustness; present the results visually (tables, tornado charts).

Chapter 6 — Financial Modeling and Cash Flow Modeling
Overview & Purpose
Explains financial modeling as the process of building a reproducible spreadsheet summary of a company’s historical financials and forward projections to evaluate future performance, cash flows, financing needs and valuation. Emphasises the three-statement model (Income Statement, Balance Sheet, Cash Flow Statement), scenario analysis (Best/Base/Worst), and discipline: mapping raw data → structured tables → formulas → schedules → linkages → scenario toggles. The chapter is a step-by-step practical guide to build a forecasting model in Excel from unarranged raw income/balance data to a full, scenario-driven three-statement forecast with supporting schedules (Fixed Assets, Debt, Equity) and a final cash flow statement.

What is Financial Modeling
Definition: spreadsheet-based forecast that projects future company financials using historical performance + assumptions.
Purpose: decision-making (growth, financing, valuation), sensitivity/scenario analysis, communicating assumptions, producing three-statement outputs for stakeholders. Highlights: mapping to historical data, structured formulas (SUMIFS/VLOOKUP/INDEX-MATCH), using Tables and named ranges, building supporting schedules, maintaining auditability.
Creating Financial Modeling — overall workflow
1 Mapping raw data to standard income statement lines
• Start with unarranged income/expense rows and map them to a standard income statement layout (Revenue, COGS, Gross Profit, Operating Expenses, EBITDA, D&A, EBIT, Interest, EBT, Taxes, Net Income).
• Insert a helper column (mapping column) that contains the standard row label for each raw row so formulas can aggregate with SUMIFS.

2 Build output Income Statement format
• Create the clean output Income Statement (historic years + forecast years columns).
• Populate historical numbers via SUMIFS from the raw table using the mapping column. Example formula pattern:
=SUMIFS(Data_of_Income_Statement!C$2:C$23,Data_of_Income_Statement!$B$2:$B$23,Income_Statement!$A4)
• Compute derived lines with straightforward formulas: Gross Profit = Revenue – COGS; EBITDA = Gross Profit – OpEx; EBIT = EBITDA – D&A; EBT = EBIT – Interest; Net Income = EBT – Taxes.

3 Map historical Balance Sheet data
• Add historical Balance Sheet table (closing balances) and map each balance line to the model’s standardized Balance Sheet labels.
Detailed Build steps and formulas
4 Add forecast columns
• Append forecast years (e.g., next 5 years) columns to Income Statement & Balance Sheet. These will be driven by scenario assumptions and derived metrics from historical averages where appropriate.

5 Create Scenario drivers
• Create driver tables for key assumptions with Best / Base / Worst cases: Revenue growth % (YoY), COGS as % of Revenue, OpEx as % of Revenue.
• For historical years, compute actual Revenue growth, COGS% and OpEx% (e.g., =ROUND((C4-B4)/C4,3) and =ROUND(B5/B4,3)).
• For scenario rows compute MAX, AVERAGE, MIN of historical metric(s) to generate Best/Base/Worst candidate values.
• Provide a dropdown selector cell (Data Validation list) that lets the model user pick Best / Base / Worst. Use VLOOKUP to pull the selected case series into the Selected Case row. Example pattern:
=VLOOKUP($D$1,$A$19:$I$21,COLUMN(),0) (explained with COLUMN() trick as COLUMN()-2 in the chapter).

6 Apply selected scenario to forecast years
• For Revenue growth in forecast years use: LastYearRevenue (1 + SelectedGrowth%).
• For COGS & OpEx in forecast years use: ForecastRevenue SelectedCOGS% and * SelectedOpEx% respectively.
• Use structured formulas and fill across forecast period.
Balance Sheet working capital metrics and forecasting
7 Calculate working-capital metrics from historical balances
• DSO (Debtors Collection Period) = Trade Receivable / Revenue 365.
• DPO (Creditors Payment Period) = Trade Payable / Revenue 365.
• DIO (Inventory Conversion Period) = Inventory / Revenue * 365.
• Other Assets % = Other Assets / Revenue; Other Liabilities % = Other Liabilities / Revenue; Provision % = Provisions / Revenue.
• Use ROUND and consistent denominator (365 or 360 per analyst choice).

8 Forecast working-capital metrics for Year1 and rolling forward
• For the first forecast year take the average of the available historical years for each metric (rolling average).
• For subsequent forecast years use a rolling-forward average approach (e.g., include most recent years and prior forecast years) so metrics evolve smoothly.
• Apply forecasted DSO/DPO/DIO to forecast Revenue to compute projected Trade Receivable, Trade Payable, Inventory: e.g., TradeReceivable = ForecastRevenue / 365 * DSO.
• Other Assets, Other Liabilities, Provisions forecast similarly as = Revenue * forecast%.
Fixed Assets roll-forward and Capex / Depreciation schedule
9 Fixed Asset schedule (FA sheet)
• Inputs: Closing Fixed Assets (historical), D&A historical from Income Statement, Useful Life or depreciation policy, Capex % schedule (Capex as % of Opening FA for forecast years).
• Key relationships:
• Opening FA (t) = Closing FA (t-1).
• Closing FA (t) = Opening FA (t) + Capex (t) – D&A (t).
• Capex (t) = Opening FA (t) * Capex% (driver).
• D&A calculation uses policy (SLM or user rate) applied to Opening FA and staggered D&A for later capex additions (chapter shows year-by-year D&A rows for capex placed in earlier years).
• Show D&A roll-forward detail lines for each cohort of capex (Year1 capex depreciates from Year2, etc.).

10 D&A % and Capex % drivers and rolling formulas
• Compute historical D&A% = D&A / Opening FA; Capex% = Capex / Opening FA.
• Use rolling average of historical D&A% and Capex% to populate forecast drivers; apply these drivers to Opening FA to get forecast D&A and Capex.
Financing schedule — Debt, interest, repayments
11 Financing sheet structure
• Inputs: Selected Case, Investment amount (initial capex), Debt principal, Interest rate, Debt term (years), Repayment schedule (% or annuity pattern), repayment % per year (e.g., equal instalments or custom schedule).
• Build month/annual repayment schedule using PMT/IPMT/PPMT functions where appropriate for annuity-style loans; chapter uses PMT to compute equal instalment and PPMT/IPMT to split principal/interest across periods. Example formulas:
• PMT(rate, nper, -pv) to get installment (rounded).
• Use IPMT for interest portion of a specific period and PPMT for principal portion.

12 Drawing power / closing debt roll-forward
• Opening Debt (Year t) = Closing Debt (Year t-1) or initial loan in Year0.
• Repayment (Year t) = Opening Debt Repayment % (or PMT principal from schedule).
• Closing Debt = Opening Debt – Repayment + New Debt (if any).
• Interest Expense = Opening Debt Interest Rate (or use IPMT per period).
• Link Closing Debt into Balance Sheet (Long-term/Current debt split according to schedule).
Equity schedule and dividends
13 Equity sheet
• Inputs: Opening equity, planned new equity injections (if any), dividend policy (e.g., 25% payout of Net Income), retained earnings roll-forward.
• Mechanics:
• Opening Equity (t) = Closing Equity (t-1).
• Increase in Capital = new injections (user input, often zero in examples).
• Net Income pulled from Income Statement.
• Dividend payout = Net Income * Payout %.
• Closing Equity = Opening Equity + New Capital + Net Income – Dividends.
• Link Closing Equity into Balance Sheet.
Income Statement forecasting (tie-ins)
14 Putting the pieces together — forecast Income Statement
• Revenue: driven by selected Revenue Growth drivers (scenario).
• COGS / OpEx: modelled as % of Revenue from selected scenario rows.
• D&A: pulled from FA schedule forecast.
• Interest Expense: pulled from Financing schedule computed interest lines (negative sign convention applied as needed).
• Taxes: calculated on EBT; tax rate applied with IF logic to avoid tax on losses (chapter uses IF(EBT>0,EBT*TaxRate,0)).
• Net Income: EBT – Taxes; this flows to Equity schedule.
Cash Flow Statement (CFS) construction
15 CFS layout and linkages
• Build a forecast Cash Flow Statement that aggregates all cash movements produced by schedules.
• Typical rows and sources: Revenue (cash collections), Opex cash payments, Taxes paid, Capex (cash outflow per FA schedule), Debt repayments (principal), Interest paid, Equity injections, Dividends paid.
• Operating Cash Flow (simplified in chapter) = Revenue – Opex – Taxes – Capex (chapter uses this simplification for operating cash flow before financing items).
• Net Cash Flow = Operating Cash Flow – Debt Repayment – Interest – Increase in Equity + any residual value receipts in final year.
• Residual / Salvage: if the model includes residual value of FA at exit, add to final year net cash flow.
16 Linking Cash back to Balance Sheet
• Closing Cash (t) = Opening Cash (t) + Net Cash Flow (t).
• Set Opening Cash for first forecast year equal to historical closing cash. Link Closing Cash into Balance Sheet cash line so the Balance Sheet balances (Assets = Liabilities + Equity).
• Reconcile totals: ensure that sum of current assets + fixed assets = sum of liabilities + equity.
17 Formulas, Excel functions & tricks used
• SUMIFS for mapping historical values into standard output rows.
• VLOOKUP used for scenario table selections; use COLUMN() trick to keep lookups portable. Example: =VLOOKUP($D$1,$A$19:$I$21,COLUMN(),0) and explanation of COLUMN()-2 when needed.
• PMT / IPMT / PPMT for loan schedules: PMT(rate,nper,-pv) and IPMT/PPMT for periodic splits.
• ROUND & ROUNDUP/ROUNDDOWN where needed to ensure polite cash figures.
• IF logical wrappers for taxes and to avoid negative taxes on losses: =IF(EBT>0,EBT*TaxRate,0).
• Use of linking across sheets (FA!, Debts!, Equity!, CFS!, P&L!) so each schedule is transparent.
18 Rolling averages and cohort accounting
• Rolling averages: use AVERAGE of historical rows to seed first forecast year; then roll forward including forecast years to smooth assumptions.
• Cohort depreciation: for capex added in forecast years, record cohort D&A rows so depreciation for each capex cohort begins in the correct later year. This preserves accurate D&A timing and closing FA balances.
19 Scenario toggles and outputs
• Model includes a top-level scenario selector (Best / Base / Worst) on the Drivers sheet tied to case driver tables.
• When users pick a scenario, Revenue Growth, COGS%, OpEx% and other rates switch automatically and the entire forecast re-calculates (Income Statement, Balance Sheet, CFS, supporting schedules).
• Chapter shows checks: generate the Income Statement, Balance Sheet and Cash Flow Statement for each selected scenario and copy outputs to review pages for Base/Best/Worst. Visual compares (later used charts) show differences across scenarios.
20 Model checks, balancing & finalisation
• Reconciliation checks: Assets = Liabilities + Equity; Net cash flow matches change in Cash; Check that Closing FA formula equals Opening + Capex – D&A.
• Sanity checks: ensure taxes are zero on losses, interest is linked to opening debt, dividends capped to Net Income when policy dictates.
• Protect calculation cells and provide a 'Model Guide' (noted practice earlier) listing assumptions, named ranges, and where inputs live.

Chapter 7 — Preparing Different Budgets With Analysis
Overview & Purpose
Explains why budgets are essential: planning revenue & costs, managing cash, controlling working capital, informing financing decisions and dividends, setting operational targets, and enabling variance analysis. Emphasizes Excel as the delivery vehicle: repeatable, auditable, scenario-driven budgets that link Sales → Production → Purchases → Collections/Payments → Cash → Financing → P&L → Balance Sheet. Stresses separation of raw data / assumptions / calculations / outputs and a strong reconciliation and audit framework.
Procedure to create a small business budget (step-by-step with Excel patterns)
Step 1 Identify Revenue
• Break revenue into logical streams (products, services, geographies).
• Forecast drivers: volumes/units × price (separate volume & price drivers).
• Model monthly seasonality: use historical month-splits, moving averages, and growth rates. Example Excel pattern: =LastYearUnits * (1 + GrowthRate) and =Units * Price.
Step 2 Determine Fixed Costs
• List monthly fixed items (rent, core salaries, insurance). Enter as constant monthly rows; use named ranges for ease of override. Document accrual vs cash timing (e.g., rent accrues monthly but paid monthly).
Step 3 Determine Variable Costs
• Link variable costs to volume (e.g., DM per unit, direct-labour per unit). Use formulas like =Units * DM_per_unit and =Units * DL_hours_per_unit * Hourly_rate.
Step 4 Identify One-time Costs
• Separate capex, restructuring, one-off marketing campaigns. Put these on CAPEX sheet and link to FA roll-forward and cash flow.
Step 5 Consolidate
• Build an operating budget table (months across columns). Sum revenue and expense categories to derive EBITDA/EBIT/Net Income. Keep calculation cells (formulas) separate from assumption cells.
Four Budgeting Methods — detailed guidance, where & how to use
Incremental Method
• Mechanic: Take last year’s line item and apply % change or fixed add-on.
• Use when: stable operations, low change environment, limited resources for re-justification.
• Excel tip: =LastYear * (1 + Increment%) with a clear assumption cell for Increment% per cost line.
Activity-Based Budgeting (ABB)
• Mechanic: identify activities → measure drivers (transactions, machine-hours) → cost per driver → budget = driver quantity × rate.
• Use when: you want to pinpoint inefficiencies or when cost structure is complex (services/manufacturing with many overheads).
• Excel pattern: use a drivers table and SUMPRODUCT or SUMIFS to allocate costs to activities.
Value-Proportion Method
• Mechanic: evaluate each budget item against value delivered (revenue, strategic goals), keep only value-add items or justify ROI.
• Use when: resources constrained and prioritization is required (e.g., marketing ROI focus).
Zero-Based Budgeting (ZBB)
• Mechanic: build every cost from zero; require manager justification, use decision units and marginal costing.
• Use when: restructuring, strong cost control required, or when legacy budgets are wasteful.
• Excel tip: use an input tab where owners enter justification and a scoring matrix that filters items into the budget.
Types of corporate budgets — roles & interlinks
Operating Budget
• Monthly projection of revenue and operating costs; feeds the P&L and the working capital schedule.
Cash Budget
• Month-by-month inflows/outflows; primary for liquidity management and short-term borrowing. Include minimum cash policy and borrowing logic.
Financial Budget
• Year-end projected Balance Sheet and financing plan; used for covenant tests, capital structure decisions.
Sales Budget
• Drives production & working capital; contains units, prices, timing (cash vs credit sales).
Production Budget
• Converts sales into production plan; includes desired ending inventory policy and raw material needs.
Labor Budget
• Headcount, hours, payroll cost; calculate both cash payroll and accruals (pay run timing).
Capital Budget
• Capex list, timing, funding; useful for NPV/IRR decisions and depreciation schedules.
Static Budget
• Fixed; used often by non-profits or entities with predictable funding.
Operating Budget — components, calculations & examples
• Revenue — monthly units × price; allow separate assumptions for unit growth and price inflation per period.
• Cost of Goods Sold — material, labour and production overheads allocated on units produced/sold.
• Operating expenses — split into variable and fixed; depreciation accounted separately as non-cash.
• Non-cash items — depreciation/amortization schedule must flow to FA roll-forward (not to cash).
• Non-operating — interest & exceptional items shown separately.
Excel worked example — architecture, sheets & naming conventions
Workbook layout
• Suggested sheets: Drivers (assumptions & scenarios), Revenue, Production, DM1, DM2, Labor, MOH, COGS, Opex, Capex, FA, Financing, P&L, CFS, BS, Equity, Checks, ModelGuide.
• Naming: use descriptive sheet names, named ranges (e.g., MinCash, TaxRate), and Table objects (Table_Sales).
• Protection: lock formula cells, leave highlighted input cells for users. Document each sheet’s purpose in ModelGuide.
Sales Budget — practical modelling patterns
• Input: monthly unit forecasts and price per unit.
• Split: cash vs credit percentages—example formula for cash sales: =Units * Price * CashPct.
• Credit collection lag: model receivables by mapping credit sales to collections in subsequent months (use offset or scheduled matching). Example: =IF(Month=Apr, CreditSales(Mar), …) or using INDEX on credit-sales range with MATCH for month lookup.
Production Budget — formulas & inventory policy
• Desired closing stock policy: e.g., DesiredClosing = UnitsForecast * ClosingStockPct.
• Production calculation: Production = UnitsToSell + DesiredClosing - OpeningStock.
• Opening stock for first month = known actual; subsequent months link OpeningStock = PrevMonth_ClosingStock.
• Excel pattern: use OFFSET/INDEX to reference previous column or arrange months in rows for simpler =B6 style linking.
Direct Material (DM) schedules — step-by-step formulas
• Required quantity = ProductionUnits * UsagePerUnit.
• Desired closing DM = RequiredQty * DesiredDMClosingPct (often 20%).
• Purchases Qty = RequiredQty + DesiredClosing - OpeningStock.
• Purchase Cost = PurchasesQty * UnitRate.
• March closing stock flows into Balance Sheet: ClosingStockValue = ClosingStockQty * UnitRate.
• Excel tip: maintain separate columns for qty and value; keep unit rates on Drivers sheet for scenario-driven price changes.
Direct Labor budget — structure & examples
• Total hours = ProductionUnits * HoursPerUnit.
• Labor cost = TotalHours * HourlyRate.
• For headcount budgeting convert hours to FTEs: FTE = TotalHours / AnnualHoursPerFTE and cost = FTE * AnnualSalary where appropriate.
Manufacturing Overheads (MOH)
• Variable MOH = UnitsProduced * VariableMOH_perUnit.
• Fixed MOH = monthly fixed lines (factory rent, maintenance).
• Depreciation on factory assets recorded separately and included in MOH as non-cash. Sum variable + fixed + depreciation = Total MOH.
COGS schedule — calculation & presentation
• Cost components per unit = DM_per_unit + DL_per_unit + MOH_per_unit.
• Total manufactured cost = CostPerUnit * UnitsProduced.
• COGS for P&L = CostPerUnit * UnitsSold. Ensure closing inventory valued at cost per unit flows to Balance Sheet.
• Excel check: ManufacturedCost - ClosingInventory + OpeningInventory = COGS (reconcile formula).
Operating Expenses & CAPEX
• Operating expense lines: split to variable vs fixed and monthly forecast each line.
• Depreciation for office equipment included under Opex non-cash D&A.
• CAPEX plan: list assets, expected spend month/year, useful life and depreciation method (SLM or WDV). Link CAPEX to FA roll-forward.
Collections & Payments calendar — timing rules & formulas
• Collections logic (example): CashSales in same month; CreditSales collected next month. Use OFFSET/INDEX to map credit sales to next-month collections.
• Payments logic (example): Supplier credit 1-month — purchases made in month t are paid in month t+1. Wages paid month-end (same month). Taxes paid quarterly/yearly — include schedule.
• Excel mapping: create monthly Cash Collection row and Cash Payment row; use SUMIFS across purchase schedules if suppliers vary.
Combined cash schedule, minimum cash policy & borrowing logic
• Combined cash position calculation pattern:
- Opening cash (from BS or prior month closing) <br> - + Collections <br> - + New Equity (if any) <br> - = Cash Available before Payments <br> - − Payments (suppliers, payroll, capex, interest, dividends, tax) <br> - = Ending cash before financing.
• Minimum cash rule: define named input MinCash (e.g., ₹200,000).
• Borrowing logic (IF pattern):
- Borrow = IF(EndingBeforeFin >= MinCash, 0, MinCash - EndingBeforeFin + PriorRepay + PriorInterest) (chapter explains including previous month repayment & interest to preserve covenants).
• Repayment scheduling: set repayment % per period or amortize with PMT. Next month closing debt = previous closing − repayment + new borrowing.
• Excel caution: avoid creating circular references unless deliberate (if you need circulars to model overdraft with variable interest, enable iterative calculation and control iterations).
Financing & loan schedule — PMT/IPMT/PPMT usage
• PMT to compute constant period payment: =PMT(rate, nper, -pv).
• IPMT for interest portion in specific period: =IPMT(rate, period, nper, pv).
• PPMT for principal portion: =PPMT(rate, period, nper, pv).
• Build amortization schedule with columns: OpeningBalance, Payment(PMT), Interest(IPMT), Principal(PPMT), ClosingBalance = Opening - Principal. Link Interest into P&L and Principal into CFS.
Income Statement budget construction — formula flow
• Sales = from Sales budget.
• COGS = from COGS schedule.
• Gross Profit = Sales − COGS.
• Operating expenses = sum of Opex schedule lines.
• EBITDA = GrossProfit − OperatingExpenses (exclude D&A).
• D&A = from FA schedule.
• EBIT = EBITDA − D&A.
• Interest = from financing schedule (link negative sign conventions carefully).
• EBT = EBIT − Interest.
• Taxes = IF(EBT>0, EBT * TaxRate, 0) to avoid paying tax on book losses (based on chapter pattern).
• Net Income = EBT − Taxes.
• Dividends applied in equity schedule and cash flow per dividend policy (e.g., 25% payout).
Fixed Assets roll-forward & depreciation mechanics
• Opening FA = last period Closing FA.
• Capex additions = direct input monthly/annually.
• D&A (SLM example) = OpeningFA * DepRate (or compute per asset for mixed lives).
• Closing FA = Opening FA + Capex − D&A.
• For new assets bought mid-year, prorate D&A (month-count) or start from next full year as applied in the worked example.
• Record revaluation or disposals separately with tax consequences if applicable.
Equity schedule — flows and formulas
• Opening Equity = prior closing equity.
• Add: New capital / share issuance.
• Add: Net Income.
• Subtract: Dividends (cash payouts).
• Closing Equity = Opening + NewCapital + NetIncome − Dividends. Link Closing Equity to BS.
Cash Flow Statement (CFS) — full build & mapping
• Operating cash flow (direct model used in chapter): Collections − Payments (excluding capex & financing).
• Investing cash flow: Capex outflows, proceeds from asset disposals or residual values.
• Financing cash flow: Borrowings (new), Debt repayments (principal), Interest paid, Equity injections, Dividends paid.
• Net cash flow = Operating + Investing + Financing.
• Reconcile CFS closing cash to Balance Sheet cash.
Budgeted Balance Sheet — assembly & line-by-line source mapping
• Current Assets: Cash (from CFS), AR (uncollected credit sales at period end), Inventory (closing production & DM stock valued at unit cost), Other current assets and prepayments.
• Non-current Assets: Closing FA (FA sheet less accumulated D&A).
• Current Liabilities: AP (unpaid purchases at period end), Current portion of long-term debt (next 12 months), accruals and provisions.
• Equity: Closing equity from Equity schedule.
• Final reconciliation check: Assets total = Liabilities + Equity. Add check cell =TotalAssets - TotalLiabilitiesEquity and flag when not zero.
Forecasting working capital drivers — detailed formulas & rolling logic
• DSO = TradeReceivable / Revenue * 365 (or use 360 if preferred).
• DPO = TradePayable / Purchases * 365 (or TradePayable / COGS * 365 depending on definition used).
• DIO = Inventory / COGS * 365 (or Inventory / CostPerDay).
• Rolling-average forecasting: use AVERAGE on historical ratios (last 3 years or last N periods) then apply to forecast revenue to derive balances: ForecastReceivable = ForecastRevenue / 365 * ForecastDSO.
• Implementation: compute historical DSO/DPO/DIO rows, then create scenario driver cells (Best/Base/Worst) populated via MAX, AVERAGE, MIN or business logic.
Scenario building & mechanics
• Create a Scenarios or Drivers table with columns for Best/Base/Worst and rows for RevenueGrowth%, COGS%, OpEx%, Capex%, DSO, DPO, DIO etc.
• Add a scenario selector cell using Data Validation (dropdown).
• Pull selected scenario values into forecast formulas using VLOOKUP or safer INDEX/MATCH or XLOOKUP. Example: =XLOOKUP(SelectedScenario, Scenarios[Name], Scenarios[RevenueGrowth]).
• For multi-year scenario stretch: either copy same scenario rates forward or model step-changes (e.g., fade from Best in year1 to Base by year3).
• Recalculate entire model automatically by linking all forecast inputs to scenario outputs.
Variance analysis & KPI reporting (budget vs actual)
• Variance absolute = Actual - Budget.
• Variance % = IF(Budget=0, "", (Actual - Budget)/ABS(Budget)).
• Favorable/unfavorable convention: for revenue positive variance is favorable; for expense positive variance is unfavorable—use sign logic or add a Favorable flag column.
• Conditional formatting rules to color variances (e.g., red for >X% adverse, green for favorable).
• Common KPIs to present: Gross margin %, EBITDA margin %, Net margin %, DSO, DPO, DIO, Working Capital / Revenue %, Cash conversion cycle (DSO + DIO − DPO), Current Ratio, Cash runway (months of min cash based on burn).
Checks, reconciliations & model validation routines (mandatory)
• Reconcile Cash: CFS_ClosingCash - BS_Cash = 0.
• Reconcile FA: FA_Closing - BS_FA = 0.
• Reconcile Debt: Financing_ClosingDebt - BS_Debt = 0.
• Reconcile Equity: Equity_Closing - BS_Equity = 0.
• Add a Checks sheet with each check and a green/red indicator formula: =IF(ABS(CheckValue) < Tolerance, "OK", "ERROR") where Tolerance = small rounding threshold.
• Add top-level Balance check: TotalAssets - (TotalLiabilities + TotalEquity) flagged when non-zero.
• Add sample sensitivity checks: change Revenue by ±10% and confirm resulting cash & covenant impacts.
Practical Excel formulas & patterns (copy-ready examples)
• Sales Revenue: =UnitsCell * PriceCell.
• COGS per unit: =SUM(DM_perUnit, DL_perUnit, MOH_perUnit).
• Purchases (DM): =MAX(0, RequiredQty + DesiredClosing - OpeningStock).
• SUMIFS example: =SUMIFS(Table_Purchases[Amount], Table_Purchases[Month], $A$1).
• Rolling average DS0 (3-year): =AVERAGE(HistDSO_Y1, HistDSO_Y2, HistDSO_Y3).
• Scenario lookup (XLOOKUP): =XLOOKUP($D$2, Scenarios[Name], Scenarios[RevenueGrowth]).
• Borrowing IF example: =IF(EndingBeforeFin > MinCash, 0, MinCash - EndingBeforeFin + PrevRepay + PrevInterest).
• Tax guard: =IF(EBT>0, EBT * TaxRate, 0).
• PMT example: =PMT(AnnualRate/12, NPeriods, -Principal) for monthly payment.
Dashboarding & reporting tips for budget presentation

Chapter 8 — Capital Budgeting and Leverage Buyout Modeling
Overview & Objectives
• Purpose: provide a full, driver-led Excel workflow for capital budgeting and an LBO case — build inputs, forecast operational and balance-sheet items, build financing and repayment schedules, prepare P&L/Cash Flow/Balance Sheet links, compute WACC/CAPM, produce DCF/NPV/IRR analyses, test scenarios, and perform LBO feasibility & acquisition pricing analysis.
• Primary outputs: time-series Revenue/OpEx/Capex/D&A, debt schedule (interest & principal), P&L, Cash Flow Statement (CFS), closing Balance Sheet items, WACC, DCF (including terminal value), NPV/IRR/MIRR, sensitivity & scenario tables, LBO implied purchase price and premium by exit year.
Model design principles
• Single source of truth: central Drivers sheet with scenario selector (Best/Base/Worst).
• Sheet separation: Drivers → Revenue → Opex → FA (Fixed Assets roll forward) → Financing (Debt schedule) → P&L → CFS → Equity Schedule → WACC → DCF → Analysis / LBO.
• Use Tables and Named Ranges for dynamic references; avoid hard-coded cell addresses.
• Keep raw data immutable; use calculation and presentation sheets separately; include a Model Guide/Assumptions sheet.
Key financial concepts & formulas
• Time Value of Money: FV and PV basics.
• NPV: ( \mathrm{NPV}=\sum_{t=1}^{N}\dfrac{CF_t}{(1+r)^t} ).
• IRR: discount rate where NPV = 0 (Excel: IRR or XIRR for irregular timing).
• MIRR: modified IRR with separate finance and reinvest rates (useful when reinvestment assumption matters).
• Profitability Index (PI) = PV(inflows)/PV(outflows) or PV(CF)/InitialOutlay.
• WACC: ( \mathrm{WACC}=\frac{E}{C}Re+\frac{D}{C}Rd(1-T) ).
• CAPM: ( Re = R_f + \beta (R_m-R_f) ).
Drivers sheet — exhaustive list
• Scenario selector: Best / Base / Worst.
• Capacity & volume drivers: Installed capacity (MW) per case, capacity factor, operating hours/day, days/year, ramp/timing assumptions.
• Price drivers: Price per unit (MWh) Year1 for each case, price escalation/inflation rate, tariff formulas or indexing rules.
• Cost drivers: OpEx % of revenue (or separate fixed & variable %), operating leverage splits, direct vs indirect costs.
• CapEx drivers: Initial investment by case, recurring CapEx % of opening FA each year, timing of capex (at Year0 or Year1), minimum maintenance capex.
• Depreciation rules: useful life by asset class, depreciation method (SL, DDB), residual/salvage value policy.
• Financing drivers: Loan amount, loan rate (fixed/floating), tenor, repayment schedule (amortizing profile or bullet), covenant terms, cushion minimum cash balance.
• Tax & accounting: Statutory tax rate, deferred tax policy, tax holiday or concession inputs if applicable.
• WACC/CAPM inputs: Risk-free rate, market risk premium, peer betas, own beta, peer leverage ratios, target capital structure, preferred / hybrid instruments if any.
• LBO specific: Current share price, fully diluted shares outstanding, current net debt, EBITDA & FCF current, debt capacity multiple (×EBITDA), exit EV/EBITDA multiple, required buyer IRR/ROE, required paydown %, optional paydown rules, minimum cash buffer.
Revenue forecasting mechanics
• Core formula (energy example): Energy MWh = InstalledCapacity × CapacityFactor × Days × Hours.
• Price evolution: Year1 from driver; Year_n = ROUND(Price_{n−1} × (1 + Inflation), 0) or use indexation rule; for multi-rate models, implement tiered pricing tables and VLOOKUP/XLOOKUP selection.
• Revenue = Energy × Price; check rounding impacts.
• Seasonality or monthly granularity: deploy monthly factors, build monthly tables and aggregate to annual; use Table columns for month/year and SUMIFS for aggregation.
Operating expenses (OpEx) modelling
• Base approach: OpEx = Revenue × OpEx% (selected case).
• Split detail: break OpEx into Variable (linked to production) and Fixed (contractual overheads).
• Escalation: apply inflation or indexed escalation to fixed cost lines separately.
• Opex timing: monthly accrual vs cash payment lag — model payables timing if working capital impact is material.
Fixed Assets roll-forward & CapEx
• Core rows: Opening FA, Additions (CapEx), Disposals (if any), D&A, Closing FA.
• ClosingFA = OpeningFA + CapEx − D&A − Disposals.
• Vintage (per-capex) schedule: for each capex vintage store cost, start-depreciation date, life and annual D&A; this enables correct D&A when dispositions/sales occur and exact residual value calculations.
• CapEx policy: Year0 initial outlay from Drivers; subsequent years capex = OpeningFA × CapEx% (driver) or a schedule of planned capex by year.
• Depreciation: straight-line example: D&A = OpeningFA × (1/useful_life) OR apply per-vintage SL with partial year conventions (half-year) if required.
Depreciation & accounting notes
• Use consistent accounting vs tax depreciation methods — add separate tax D&A lines if required for deferred tax tracking.
• Partial year: implement prorata D&A for acquisition year (days/365) if accuracy matters.
• Residual value: model per-asset salvage assumptions and include proceeds in final year's cash flow or on disposal.
Financing — debt schedule mechanics
• Inputs: principal, drawn date, interest rate (fixed/floating), amortization profile (% or constant PMT), fees, covenant triggers.
• OpeningDebt_{t} = ClosingDebt_{t−1}; Repayment_{t} = schedule% × OpeningDebt_{t} OR PPMT from PMT schedule.
• Interest_{t} = OpeningDebt_{t} × rate (adjust for day-count if monthly/quarterly).
• ClosingDebt_{t} = OpeningDebt_{t} − Repayment_{t} + NewBorrowing_{t} (if any).
• Use PMT/IPMT/PPMT for level payments or explicit principal schedule for bespoke amortization.
• Fees and capitalised interest: model separately and capitalise if debt draws and project construction warrants.
Debt covenant modelling & tests
• Typical covenants to model: Leverage ratio (NetDebt/EBITDA), Debt service coverage ratio (DSCR = EBITDA / (Interest + principal due)), Interest coverage (EBIT/Interest), Minimum cash balance.
• Implement calculations and conditional flags (TRUE/FALSE) and create “covenant breach” alerts and remedial actions (e.g., equity top-up, waiver fees, curtail capex).
Interest & capitalisation rules
• Interest payment timing: accrue monthly and pay monthly/quarterly as per loan terms — choose convention consistently.
• Capitalisation: during build/construction, interest may be capitalised into FA; after commercial operation date, expensed. Model capitalised interest line in FA roll-forward when applicable.
Profit & Loss statement linkages
• P&L rows: Revenue → Gross profit (if COGS exists) → OpEx → EBITDA → D&A → EBIT → Interest → EBT → Tax → Net Income.
• Link revenue from Revenue sheet, OpEx from Opex, D&A from FA, Interest from Financing sheet.
• Taxes: Tax = MAX(0, EBT) × TaxRate; model tax carryforwards separately if using loss utilisation rules.
Equity schedule & dividends
• Opening Equity = prior closing; Add: Net Income, New Equity injections; Less: Dividends, Share repurchases.
• Dividends = DividendPayout% × NetIncome when NetIncome>0 (implement IF condition).
• Closing Equity flows to Balance Sheet; reconcile Retained Earnings line with accumulated Net Income minus dividends.
Cash Flow Statement (CFS) — construction & reconciliation
• Structure: Operating Cash Flow (OCF) — Investing Cash Flow (CapEx, disposals) — Financing Cash Flow (debt draws/repayments, dividends) = Net Change in Cash.
• OperatingCF can be built as NetIncome + D&A +/- Working Capital changes (if model includes WC).
• For capex-heavy projects, present both Free Cash Flow to Firm (FCFF) and Free Cash Flow to Equity (FCFE) as separate rows.
• ClosingCash_{t} = OpeningCash_{t} + NetChangeInCash_{t}; ensure ClosingCash appears on Balance Sheet and ties.
Free Cash Flow definitions
• FCFF (unlevered): FCFF = EBITDA − TaxesPaid (on EBT adjusted) − CapEx − ΔWorkingCapital + Non-cash adjustments.
• FCFE (levered): FCFE = FCFF − Interest*(1−Tax) + NetBorrowing − PrincipalRepayments − PreferredPayments.
• Chapter uses FC based on Operating Cash Flow minus CapEx and debt repayments as appropriate for DCF inputs.
WACC & CAPM detailed calculation
• Inputs: Risk-free rate Rf, Market risk premium (Rm − Rf), peer/unlevered beta, tax rate, target or actual debt/equity proportions.
• Unlevered beta: β_unlever = β_peer / (1 + (1 − Tax) × (D/E)).
• Re-lever beta: β_relever = β_unlever × (1 + (1 − Tax) × (D/E_project)).
• Cost of equity: Re = Rf + β_relever × MarketRiskPremium.
• Cost of debt: Rd = coupon rate or market yield on new debt; use pre-tax cost then apply (1−Tax) in WACC.
• Weighting: use market values (preferred) or book values if market is not available. Document choice.
Discounted Cash Flow (DCF) approaches & terminal value
• DCF timing: discount FCFF by WACC (firm value) or FCFE by cost of equity (equity value).
• Terminal value methods: Exit multiple (EV_exit = EBITDA_exit × exit_multiple) OR Gordon Growth: TV = FC_{N+1} / (r − g) where g is perpetual growth; choose method and justify.
• Present value: Discount CFs and TerminalValue to PV using WACC or cost of equity depending on chosen FCF type.
NPV, IRR, MIRR and interpretation
• NPV: compute in Excel with =NPV(rate, range_year1_to_N) + initial_outlay (negative).
• IRR: =IRR(range_including_year0) — remember multiple IRR risk for nonnormal cash flows; inspect NPV profile.
• MIRR: use finance and reinvestment rates for realistic return estimates. Excel: =MIRR(range, finance_rate, reinvest_rate).
• Investment decision rules: NPV>0 and IRR>hurdle rate (but prefer NPV for mutually exclusive projects).
Common DCF pitfalls & validation
• Do not include Year0 in NPV() range; add initial outlay separately.
• Verify sign convention consistently (inflows positive, outflows negative).
• Check multiple IRRs when cash flow signs change; consider MIRR or NPV profile instead.
• Reconcile DCF result to balance sheet and check implied multiples relative to peers.
Scenario analysis & sensitivity testing
• Scenarios: implement Best/Base/Worst with driver sets; switch via drop-down using VLOOKUP/XLOOKUP/INDEX-MATCH.
• Sensitivity tables: build one-way and two-way Data Tables for price vs volume, price vs discount rate, etc.
• Tornado chart: rank sensitivities by NPV impact to show top 10 drivers.
• Use goal seek for single variable breakeven (e.g., price that yields NPV=0).
LBO modelling — drivers & initial setup
• LBO Drivers: purchase currency & method, assumed revenue growth, EBITDA & FCF margins, LBO debt multiple (×EBITDA), exit multiple, required buyer return (target IRR/ROE), minimum cash balance, annual required paydown%, optional paydown rules, transaction fees, refinancing assumptions, equity rollover percentage (if applicable).
• Tools sheet: compute MarketCap = SharePrice × FD_shares; EV = MarketCap + NetDebt; current EV/EBITDA; use these to sanity-check acquisition multiples.
LBO cashflow mechanics & debt waterfall
• Opening LBO Debt Year1 = DebtCapacity × CurrentEBITDA (or drawn debt from acquisition financing).
• Yearly sequence: Start Cash → Add FCF → pay required paydown → compute optional paydown (cash available after preserving minimum cash) → pay optional paydown up to outstanding debt → end Cash → compute Closing Debt = OpeningDebt − (required + optional paydown).
• Interest accrual: Interest on OpeningDebt; cash interest payment reduces Cash unless capitalised.
• If revolver or working capital facility included, model draw/repay as part of waterfall.
LBO purchase pricing & returns calculation
• Exit EV = Exit EBITDA × ExitMultiple.
• Exit Equity Value = Exit EV − NetDebt_at_exit.
• Required initial equity to achieve target ROE/IRR: Equity_needed = ExitEquityValue / (1 + RequiredReturn)^n OR solve iteratively for equity investment that yields target IRR given interim cash flows to equity holders.
• Alternatively compute implied purchase price per share: ImpliedPrice = (Equity_needed − CurrentNetDebt) / FD_shares depending on conventions — careful with signs.
• Premium% = (ImpliedPrice / CurrentSharePrice) − 1.

Overview & Purpose
Why dashboards: compress transactional detail into one-window visual reports that support quick decisions, monitoring, alerting, and ad-hoc exploration. Dashboards reduce cognitive load, surface anomalies, support story-telling, and provide single-source-of-truth snapshots for stakeholders.
Primary goals: accuracy, refreshability, clarity, interactivity, reproducibility, auditability, performance, and accessibility.
Key principle: build dashboards from clean, structured source data (Excel Tables or a data model). Prefer query-driven ETL (Power Query) and an analytical engine (PivotTables, Power Pivot / DAX) rather than sprawling cell formulas. Maintain separation: Source (raw) → Engine (transform + model) → Presentation (dashboard sheet).
Audience alignment: define primary users (executive, manager, analyst) and tailor granularity, visuals, and navigation accordingly. Include a clear question the dashboard answers and the KPIs that align to decisions.
High-level creation workflow
1 Data selection & prep — ingest raw data into Power Query or into an Excel Table; perform tidy transformations; validate row counts and types; build surrogate keys where needed.
2 Design & layout — wireframe on paper or digital canvas: primary KPI area, time-series area, breakdowns (rankings/segmentations), detail table, and filter panel (slicers/timelines). Define mobile/print-safe zones if needed.
3 Build data engine — create calculated columns, measures (DAX) or helper columns; define consistent period keys and calendar table; establish relationships in Data Model for multi-table scenarios.
4 Visuals — add PivotCharts, charts from measures, KPI tiles, numeric cards, tables, and sparklines; maintain consistent chart types for similar comparisons.
5 Controls — add slicers, timelines, form controls, or parameter tables; connect controls to model/pivots and define default states.
6 Polish & test — format typography and colors, align objects, lock layout grid, test combinations of filters and extreme cases, reconcile totals to source, and run performance checks.
7 Secure & publish — protect display sheet while allowing control interaction, document refresh steps, save versioned file, and publish to secure shared location or migrate to Power BI for web distribution.
Selection of data (deep)
Data shape & tidy rules: one record per row; atomic columns; no merged cells; single value per cell. Columns: Date, PeriodKey, Region/Store, ProductID, ProductName, SalesPersonID, TransactionID, Quantity, Value, CurrencyCode, Cost, Discount, Channel, SourceFileID.
Minimum recommended columns: Date, PeriodKey (YYYYMM), FiscalYear, FiscalPeriod, Region, Store, Product, SKU, Salesperson, TransactionID, Units, GrossSales, NetSales, Cost, Margin, Currency.
Data validation rules: date parseability, numeric ranges, required fields not blank, referential integrity for keys, currency consistency, duplicates detection.
ETL transforms: trim, remove control characters, change-type enforcement, parse text dates, split concatenated fields, unpivot/pivot as required, merge lookups using left joins, deduplicate, fill down/up for hierarchical attributes, calculate extended attributes (e.g., Price = Value/Quantity), and add period columns: Year, Quarter, MonthName, MonthNumber, WeekNumber, ISOWeek, DayOfWeek.
Time/calendar tables: always create a dedicated Date table with continuous range, flags for business days, fiscal period mapping, holiday flags, and attributes used for slicers (MonthName sorted by MonthNumber, FiscalYear, QuarterLabel). Mark it as Date Table in Power Pivot for DAX time intelligence correctness.
Tables vs Data Model rules: use Excel Table + Pivot for single-table reporting of small-to-medium datasets (<1–2M rows depends on Excel/desktop capabilities). Use Power Query + Data Model (Power Pivot) when multiple tables, large datasets, repeated refreshes, or advanced time intelligence are required. Consider using SQL/warehouse upstream for very large data.
Convert to Excel Table & naming
Convert: use Ctrl+T or Insert → Table for source ranges to ensure auto-expansion and structured references.
Benefits: dynamic range expansion, structured references that simplify formulas, compatibility with Power Query and PivotTables, and auto-detection by Data Model import.
Naming conventions: adopt a consistent, descriptive naming standard, e.g., tbl_Raw_Sales, tbl_Dim_Product, tbl_Fact_Transactions. Use Name Manager and Table Design → Table Name to set names. For pivots: pt_MonthSales, pt_StoreRanking; for charts: ch_MonthTrend; for slicers: slicer_Product.
Metadata: store source metadata (source path, last refresh datetime, row count, schema hash) in a control sheet for auditing.
Pre-visualization design (layout & UX)
Sketch first: create multiple wireframes: executive summary, manager drill-down, and analyst view. Map KPI to primary decisions and list interactions and drill paths.
Visual hierarchy: place most-critical KPI(s) top-left or top-center. Use size to denote importance: larger area for trend charts, medium for breakdowns, compact for details.
Navigation patterns: tabbed view (multiple dashboard sheets) for different audiences; top navigation bar with buttons (hyperlinks or macros) for switching; "back to summary" link from drill sheets.
Spacing & alignment: align to an invisible grid, use consistent padding and margins; snap objects to cells for stability across screen sizes; standardize tile sizes for visual rhythm.
Color & branding: define a limited palette (primary, secondary, neutral, alert). Use neutral greys for backgrounds and accent colors for positive/negative or highlight; maintain color-blind safe combinations; avoid gradients and 3D effects. Document color hex or RGB in a legend sheet.
PivotTables as the dashboard engine
Create: Insert → PivotTable → choose Table/Data Model. For Data Model pivots, use Insert → PivotTable and select "Use this workbook's Data Model".
Field layout patterns: Rows = primary category (region, product), Columns = time or secondary dimension, Values = measures (Sum, Count, DistinctCount where needed). Use Value Field Settings to change aggregation and Show Values As for % of total or running totals.

Grouping: use date grouping (Months, Quarters, Years) or numeric binning for histograms. When grouping months, ensure sorting uses MonthNumber key.

Pivot Calculated Fields vs Measures
Calculated Fields (classic Pivot): operate on row-level input fields, recomputed post-aggregation which can produce misleading ratios; keep simple arithmetic only and avoid for time-intel or context-dependent logic.
Measures / DAX in Power Pivot: use for robust aggregations, correct ratio calculations, and advanced time intelligence. Examples: TotalSales := SUM(Fact[NetSales]), GrossMargin := DIVIDE([TotalSales] - SUM(Fact[Cost]), [TotalSales]), SalesYoY := CALCULATE([TotalSales], SAMEPERIODLASTYEAR('Date'[Date])).
Distinct Count & advanced aggregations: prefer Data Model measures for DistinctCount, Weighted averages, and complex business logic.

Performance optimization
Data volume strategies: aggregate upstream in Power Query or source DB; use incremental refresh patterns where supported; import needed columns only and remove unused columns; filter historical data if not required.
Avoid volatile functions: refrain from NOW, TODAY, OFFSET, INDIRECT, CELL("address") in heavy models; they cause full workbook recalculation.
Minimize cell-level formulas: prefer measures and pivot aggregations over thousands of SUMIFS/INDEX/MATCH formulas in the presentation layer.
Pivot cache & model tuning: reduce duplicate pivot caches, compress workbook by removing unused connections and tables, use binary file format (.xlsb) for size reduction when macros present.
Power Query performance: push transformations as early as possible, avoid row-by-row transformations, fold queries to source for pushdown, and use Table.Buffer cautiously to prevent repeated recalculation.
