Resolution (conceptual, exact places to change, tests & expected outputs)
Symptom PTT output pages lack captions that TV shows. Root cause: key mismatch and timing — TV’s wrapper matches more key variants and runs after fragment composition; PTT relies on a narrower lookup and may remove headings before client-side wrapper runs.
Goal Same caption appears in final HTML whether produced by PTT server writer or by TV wrapper. Caption must be sanitized, not duplicated, and accessible (<caption> preferred, H3 for UI).
Principle (high level) Centralize label lookup and normalization. Inject caption at the final composition point (writer-time) and still allow TV wrapper to supplement but not duplicate. Always escape/sanitize label text.
Core levers (what can be changed) 1. Label lookup algorithm (normalize keys).
2. Where injection happens (writer-time vs wrapper-time).
3. Injection format (<caption> inside table + H3 above table).
4. Duplication guard (detect existing <caption> or server caption H3).
5. Diagnostics/logging.
Shared helper (single source of truth) render/core_utils.py — add/export find_label_for_key(key: str, out_dir: Optional[Path]=None) -> Optional[str].
Behavior: load labels.json from candidate paths, produce prioritized normalized variants (exact, lower, underscore/hyphen, compact, numeric-normalized prefix_01, prefix1, table_01, tbl_1, nested dict lookups like book: { "01": ... }, fuzzy numeric-suffix matches).
Return first match and record attempted variants via _record().
Server-side change (writer-time) File: render/core_renderer.py — inside writer(f) when writing fragments (the non-fallback branch):
1) Compute candidate keys in order: (a) book_prefix_{idx:02d} (b) fragment title_id (c) title_text or derived data-table key.
2) Call find_label_for_key() for each until match.
3) If match → produce safe_cap = html.escape(match) then: if fragment contains a <table> and contains no <caption> insert <caption>safe_cap</caption> immediately after the first <table ...> open tag.
4) Also insert an H3 server caption above fragment if no existing header with identical text.
5) Log the key tried and result.
Rationale: writer is last server composition step; this guarantees final HTML contains caption even if client wrapper later modifies DOM.
Client-side wrapper change (TV) File: html_renderer.py — keep current _find_label_for_key() (or import the shared helper).
When building wrapper: try table_id and also data_table_key derived from H3 text; prefer labels; if found and table lacks <caption> insert <caption> inside the table and/or produce the H3 above if not present.
But treat writer injection as authoritative: wrapper must skip duplicate injection if it detects <caption> or server H3.
Duplication guards Rules:
1) If table has <caption> already, do not add another.
2) If previous slice (header) exists and its stripped text equals caption text, skip adding H3.
3) If server has already added H3 (class ptt-server-caption or table-caption), skip.
Sanitization & security Always html.escape() label text before embedding.
If sanitize_html() available, pass escaped string through sanitizer with allow_tables=False.
Never interpolate raw label text.
Diagnostics Add _record() calls: record lookup attempts with variants, matches, and misses.
Expose in get_import_diagnostics() or logs.
Example messages: "label_lookup tried: ['Table1','table_01','table01'] -> match=table_01".
Backwards compatibility Existing clients expecting legacy ids not broken: do not change table id attributes.
H3 inserted above table must not carry the same id as table.
Legacy alias elements (copyAllMdBtn) remain untouched.

Table 3
System
Caption Injection Model
Tables Viewer (TV) Injects captions after rendering. Uses BeautifulSoup to scan <table> blocks, detect missing captions, read metadata, then insert <caption> or <h3> before output. Logic is tolerant: finds targets by tag shape, attributes, and structural clues.
PasteToTable (PTT) Renders HTML directly in html_renderer.py without a BS4 post-pass. Caption must be generated at render time from metadata delivered through core_renderer.py. No DOM correction layer.
