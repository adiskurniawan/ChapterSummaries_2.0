Logic / Structure
Conceptual notes, failure modes, and diagnostic checkpoints
Overall goal
 script.list.js builds ordered sources for a group. core_renderer.py must receive sources and inject captions so UI order and metadata match. Captions missing means ordering mismatch, detection failure, or silent server exceptions.
script.list.js
_buildSourcesMeta(list)
 Produces {name, path, url, caption, description} per file. Check every entry for completeness and type safety. Common errors: nulls, empty captions, or unreachable URLs. Log sourcesMeta before _notifyServerRender.
script.list.js
_notifyServerRender(text, sources) → POST /api/render
 Sends {text, sources, fromList:true}. Risks: wrong endpoint, CORS rejection, malformed JSON, or timeout. Client doesn’t block; result stored in window.__ptt_server_render_result. Validate POST payload, response, and that property.
Server contract
 Server must forward sources into render_html(..., sources=sources). If ignored or renamed, captions never appear. Confirm request handler uses the right key.
core_renderer.render_html(..., sources=...)
 Checks sources truthiness and list type. If empty or None, legacy caption logic runs. Log len(sources) and type early.
render_html → fragments_map
 Built by render_fragments_concurrently. Captions rely on positional parity between fragments and sources. If counts differ, mapping fails. Compare lengths of sources, frag_paths_ordered, and fragments_map.
frag_paths_ordered construction
 Iterates sorted(fragments_map.keys()) to create (idx, path, title_id, title_text). Index order may not match client expectations. Verify sorted keys, stems, and index continuity.
inject_captions_into_html(html_text, sources, title_ids, labels_index, book_prefix)
 Splits HTML by <div class="table-wrapper"...> and injects captions sequentially. Assumes exact count alignment between wrappers and sources. Mismatch causes offset captions or omissions. Add defensive checks and logging.
_get_caption_from_sources(item, labels_index)
 Resolves caption by caption → description → name/path. Falls back to labels_index if available. Failure when caption empty string vs None, HTML-marked text, or label normalization mismatch. Print resolved caption per source for verification.
Ordering assumption
 inject_captions_into_html inserts captions in sources order matching .table-wrapper sequence. If the server reordered or filtered fragments, captions misalign. Most likely cause: sources count/order ≠ number/order of wrappers. Compare #captions_nonempty vs wrapper count.
HTML splitting fragile points
 Regex (<div\s+class=["\']table-wrapper["\'][^>]*>) only matches exact class="table-wrapper". It fails for multi-class wrappers (class="table-wrapper extra"). When split fails, injector falls back to <table class="tv-table"...>. If both fail, HTML unchanged. Adjust regex for class-list tolerance.
Existing caption detection (skip_inject)
 Function checks if preceding out_parts[-1] ends with caption markup. False negatives cause duplicates; false positives skip valid injection. Inspect fragment HTML around wrapper to confirm caption structure.
Title id usage (title_ids param)
 Uses title_ids to set caption ids. If lists differ in length or order, wrong ids applied. Validate title_ids vs actual fragment id="Table\d+" attributes.
data-table generation inside injector
 Injector builds data-table using book_prefix and path/name. Mismatch arises if fragments already contain their own data-table identifiers. Compare produced vs existing attributes.
Fallback splitting on <table class="tv-table"
 Fallback only matches exact class equality. If table markup contains extra classes, split fails and captions vanish. Search HTML for class variants to confirm.
Sanitization / escaping
 _get_caption_from_sources escapes text. If caption already HTML-escaped, result double-escaped. Check raw captions vs final output.
Skip injection when all captions empty
 If all resolved captions empty, injector returns original HTML. Log captions array and verify nonempty entries exist.
frag_paths_ordered vs sources mismatch
 When lengths differ, some captions unused or missing. Legacy caption path disabled if client_captions_provided true. Compare both counts and build a mapping table for diagnosis.
need_fallback true path
 Fallback renders inline captions using labels.json. If client_captions_provided false or server ignored sources, fallback triggers unexpectedly. Verify need_fallback condition and logs.
Race conditions & concurrency
 render_fragments_concurrently may not finish writes before caption injector reads. Confirm all fragment files exist before injection stage.
Silent exceptions & suppressed logs
 Errors often swallowed or logged with logger.exception() but not surfaced. Escalate logging level or insert diagnostic markers into HTML to confirm injection run.
labels.json fallback behavior
 Server _labels may differ from client labels index. Missing or mismatched labels cause blank captions. Verify server _labels keys and content.
Client → server mapping mismatch (probable)
 Common causes:
1. sources order differs from render order.
2. Regex mismatch.
3. All captions empty.
4. Handler not passing sources.
5. title_ids misaligned.
6. skip_inject misfires.
Check in this order.
Non-code debugging checklist
 1. Inspect sourcesMeta on client.
2. Confirm POST body and server receipt.
3. Log sources at server entry.
4. Compare len(sources) vs wrapper count.
5. Log fragment keys and order.
6. Print counts and mappings before injection.
7. Verify _labels load.
8. Confirm injection produced modified HTML.
Conceptual fixes (design only)
 A. Match fragments to sources by identifier (data-path or data-name), not index.
B. Make regex class match tolerant.
C. Retain legacy captions until at least one injected.
D. Suppress duplicates only after verifying injection success.
E. Add diagnostic HTML comments to confirm caption attempt.
Edge cases
 Multiple tables per fragment.
Existing captions with variant markup.
Unequal counts.
HTML in captions.
Title collisions.
Similar class names.
Matching heuristics & labels interplay
 Align client and server label sources. Prefer client-supplied captions when available to prevent divergence.
Minimal safe behavioral change
 Implement idempotent injection: try data-path or data-name match first, then fallback regex. Preserve legacy captions if no insertion occurred.
Mapping table for debugging
 Log per-row:
index / source.name / source.path / caption / frag_key / frag_path / table_count / snippet. Reveals alignment failures.
Suggested test scenarios
 1. One table single file.
2. Multi-file with missing caption and label fallback.
3. Dedup reducing fragment count.
4. Multi-table fragment.
5. Wrapper with multiple classes.
Verify caption correctness.
Quick failure indicators

window.__ptt_server_render_result missing.
Server logs “caption injection failed.”
Priority to fix root cause
 1. Confirm handler forwards sources.
2. Compare counts.
3. Implement identifier-based matching.
4. Make regex flexible.
5. Keep legacy captions until injection verified.
6. Add temporary diagnostic logging.

Table 2
Step
Details
Objective
 When user clicks “Load Group” the client must send the active labels.json payload alongside the group file list to the server conversion endpoint so convert.py can inject captions server-side.
Constraints & assumptions
 labels.json is a simple mapping of ID→caption. No frontend parsing beyond loading and forwarding. Backward compatibility with existing convert endpoints must be preserved.
High-level flow
 User clicks Load Group → client collects group file descriptors → client loads current labels.json → client POSTs a single JSON request containing file list + labels → server convert.py consumes labels and injects captions during conversion → server returns rendered output.
Frontend change (behavior)
 On Load Group, load labels.json (if not cached). Attach contents as a field named labels in the conversion request payload. Do not modify file content client-side.
Frontend change (efficiency)
 Cache labels.json in memory for the session. Re-fetch only if cache miss or explicit reload. Keep payload small by sending only keys present in selected group when feasible.
Transport contract
 Single POST JSON object. Include: file descriptors or paths, metadata identifying source, and labels object. Server must accept absent labels (backward compatibility).
Server change (API handling)
 convert.py must read incoming JSON, extract labels safely, and pass it into caption-injection logic as an authoritative lookup table for file IDs. If labels missing, use existing behavior.
Injection timing & location
 Inject captions at the final content assembly stage, immediately before markdown→HTML or HTML finalization. This avoids disrupting parsing and sanitization steps.
Matching strategy
 Use exact ID lookup first. If not found, do not invent captions. No fuzzy matching. Keep injection deterministic and idempotent.
Formatting policy
 Server chooses safe caption insertion format. For Markdown outputs prefer a stable heading level prefix. For HTML outputs prefer dedicated caption element outside table cells. Do not alter original cell content.
Validation
 Validate labels is an object/dictionary with string keys and string values. Reject or ignore anything else. Log but do not fail conversion on invalid label entries.
Security & sanitization
 Treat labels as untrusted input. Escape HTML and sanitize content on server before insertion. Disallow HTML tags or scripts inside captions. Enforce maximum caption length.
Error handling & fallback
 If label lookup or injection fails for a file, continue conversion without caption for that file. Report per-file failures to the conversion log only when debug is enabled. Return a non-blocking warning in response metadata.
Performance considerations
 labels.json is small. Network overhead is negligible. If many groups are loaded in a batch, consider sending only relevant label keys to reduce payload.
Logging & observability
 Emit structured logs: request id, number of files, labels count, injection success/fail per file. Include metrics for latency added by label processing.
Testing matrix
 Unit tests for convert.py injection function. Integration tests: client sends labels; server injects captions in Markdown and HTML outputs; missing labels should be silent. Regression tests to ensure existing endpoints still work without labels.
Backward compatibility
 Server must accept requests without labels. Client must tolerate server that ignores labels. Use feature flag if phased rollout is required.
Deployment & rollback
 Deploy server changes behind a feature flag. Canary with small set of users. If injection causes issues, toggle flag and roll back quickly.
Operational checklist
 Add schema validator for labels.json. Add monitoring alert for injection error rate. Add an admin/maintenance path to replace or prune labels if needed.
Documentation & user guidance
 Document payload contract and caption rules. Provide short write-up: where to edit labels.json, expected ID format, and how captions will appear in outputs.
Acceptance criteria
 Clicking Load Group posts files + labels. Server uses provided mapping to inject captions for files with matching IDs. No client-side content alteration. Conversion succeeds and captions appear in final outputs for matching IDs. Logs show injection activity.
