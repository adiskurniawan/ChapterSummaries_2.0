Logic / Structure
Conceptual notes
Live file discovery
 Probes multiple server endpoints (LIVE_FILES_ENDPOINTS) sequentially to find a directory/list representation. Normalizes results to {name,url,path,description}.
Labels and groups
 Loads labels.json and groups.json candidates. Builds a robust label lookup (buildLabelIndex) and group descriptions for display and caption resolution.
Caption resolution
 resolveCaptionFor(baseName, labelIndex) generates many canonical variants and picks the longest/specific match. Used when building groups from live files.
Group building
 buildGroupsFromLive maps files into groups using groupKeyFromPath. Attaches caption for each file (explicit description → resolved label → empty).
UI
modal + group rows
 presentGroupsOnly builds modal with persistent DOM rows. Each group row has a Load button and file rows that call loadFilesSequentialAndHideForm(single) when clicked.
File loading flow
 loadFilesSequentialAndHideForm fetches each file sequentially, produces accRaw (raw content) and accMarked (with injected HTML comment caption + markdown heading). Sets window.__ptt_runtime_captions and calls client convert hooks (window.__tv_do_convert / window.convert) or fallback hidden textarea.
Runtime caption application
 After convert, applyRuntimeCaptions repeatedly searches for caption nodes (.table-caption or h1..h6) and applies runtime captions. Uses retries.
Search engine
 Local file-level inverted index and scoring for searching within groups/files.
Copy / clipboard helpers
 Delegates to renderer functions (copyTableMarkdown, copyTablePlain) when available. Fallback to raw paste/markdown extraction otherwise.
Defensive coding
 Timeout-safe fetch wrappers, BOM stripping, HTML escaping, many try/catch guards.

Logic / Structure
Conceptual Notes
HTML composition (head/tail)
 Builds <script> and <style> bundles dynamically. Embeds worker export logic and visibility triggers for .table-wrapper. Final HTML assembled via head_parts + tail_parts.
_remove_heading_from_fragment()
 Defensive regex-based scrubber. Removes <h1–h6> tags matching known title_id or title_text from fragment content to prevent duplicate headings after caption injection.
_build_caption_payload_for_fragment()
 Central function to extract and normalize caption metadata per table. Handles multiple schema variants: meta["captions"], caption_payload, captions_json, or caption. Sanitizes text, enforces deterministic IDs (book_prefix_idx), and sorts payload entries lexically. Produces safe, flat list for JSON embedding.
writer(f) closure
 Main rendering loop. Writes head → TOC → optional banner → tables → tail. Uses allow_inject from _allow_caption_injection_from_cfg(cfg) (derived from request context).
logger.debug trace
 Logs injection allowance and current request context (cfg.action, cfg.source, etc.) for debugging injection behavior.
Caption + wrapper creation
 For each fragment, builds .table-wrapper div with caption div. Generates caption text hierarchy: _caption_for(book_prefix, idx) → title_text → fallback "Table <n>". Each wrapper receives a unique id and data-table attribute.
Payload generation & embedding
 If allow_inject is true, or _injection_allowed_override exists, builds caption payload via _build_caption_payload_for_fragment. Emits <script type="application/json" data-caption-payload> immediately below the caption. Adds optional data-client-req-id if request_ctx carries it.
Injection marking
 Marks fragment meta as injected=True and adds "injection_marker":"core_renderer". Guarantees idempotency: prevents re-injection on repeated renders.
Fallback writing logic
 If no payload, writes <div class="table-core"></div>. If need_fallback true, dumps inline HTML instead of structured tables.
Atomic write safety
 Uses _default_write_atomic if available; else manually writes to .tmp file and replaces atomically. Ensures data integrity under concurrent renders.
Exception isolation
 Wraps every stage in try/except to preserve downstream output even on fragment-level failures.
Result dictionary
 Returns structured JSON summary: warnings, tables, rows, runtime_ms, output, optional html. Compatible with frontend logging or post-render validation.

Conceptual notes
Purpose
 Thin compatibility layer and post-processor for the core render functions. Wraps render_html, render_table, render_page if available; provides fallbacks when render modules missing.
Configuration
 _renderer_config driven by env + set_renderer_config. Controls ensure_title, minify_html, table_ui_css/js, table_ui_max_attr_json_len, and path processing.
Post
processing wrapper (_wrap_callable)
 If wrapped function returns str or bytes or Path, it applies: ensure-title, minify, UI asset injection, table wrapping. Returns original output type where possible.
Title enforcement
 _ensure_title_in_h1_fragment / _ensure_title_attrs_in_str add class and id to the first <h1> to meet accessibility and anchor expectations.
Minification
 _minify_html_str tokenizes and collapses whitespace while preserving script/style/pre/code/textarea sections. Enabled by config.
UI asset injection
 _inject_ui_assets ensures table_ui_css and table_ui_js are present. Inserts <link> in <head> or <script defer> before </body>. Detects by basename to avoid duplicates.
Table wrapping (_wrap_each_table_with_ui)
 Scans for <table> blocks. For each table it: preserves cell newlines, locates a preceding heading or .table-caption, ensures or generates stable table_id, builds table-wrapper HTML with data-ptt-json or emits <script type="application/json" data-ptt="TableId"> if JSON too large. Skips wrapping if a table-wrapper already encloses the table.
JSON
in-attribute threshold
 Uses table_ui_max_attr_json_len to decide whether to inline JSON in data-ptt-json attribute or emit a separate <script> payload.
Protected content handling
 Preserves code/pre/textarea/script sections during minify and newline conversion.
Fallback renderers
 When render.core_compat not available, provides _fallback_render_html, _fallback_render_table, _fallback_render_page that produce simple HTML and call _wrap_callable for post-processing.
Diagnostic helpers
 get_import_diagnostics, generate_diagnostic_report, and _record collect import attempts and processing notes for debugging.
Embedded create_app
 Fallback Flask app with /api/convert and /api/render. Uses wrapped render_html if present. Protects request size. Sets minimal security headers.
