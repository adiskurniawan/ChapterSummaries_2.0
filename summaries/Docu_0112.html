Area
PTT Implementation Summary
Purpose Unified front-end combining PasteToTable and Tables Viewer. Allows pasting Markdown or table text, converting it client-side, and exporting or saving it.
Structure Full standalone HTML5 document with complete <head> and <body>; designed as root index.html entrypoint.
Header (UI) Sticky toolbar with buttons for Export All, Save, Save Markdown, Toggle All, Clear, Load Tables, and Convert. Includes inline paste box (fixed 8 cm width) and search field.
Styling Inline base CSS defines theme variables, typography, sticky header, responsive behavior, and normalized .tv-table class. External style.css and overrides.css extend it.
Accessibility Proper ARIA roles and labels for all key sections (role="banner", role="toolbar", role="main", etc.).
TOC Handling Merged handling for both top and bottom TOC containers (#toc and #tocBar), unified style and logic via script.toc.js.
Script Load Order Strict sequential loading: xlsx.full.min.js → script.toc.js → script.core.js → script.table.js → script.save.js → script.list.js → worker.js → script.js. Prevents race conditions.
Interactivity Inline scripts handle: alias delegation between copyAllTablesBtn and legacy copyAllMdBtn; dynamic “Save” logic (custom event ptt:save-request); Markdown save dispatch (ptt:save-md-request); TOC and caption injection; MutationObserver reveal on DOM change.
Save Logic Robust multi-path handler: prefers window.PasteToTable.save() if defined; falls back to saveTables() or direct POST to /save. Includes promise handling, live status updates, and safe fallback messages.
Resilience All inline JS functions are wrapped in try/catch; no blocking errors on missing elements.
Progress vs TV build - Adds full Paste and Convert UI integrated in header.
- Merges Tables Viewer and PasteToTable flows into one surface.
- Contains caption injection and TOC bootstrap inline instead of separate loader.
- Fully responsive with small-screen title suppression.
- Adds markdown-save dispatch and API fallback for /save endpoint.
Operational State Ready for production deployment; all external assets versioned via v=%%ASSET_HASH%%.
Overall Difference from TV PTT acts as entrypoint application, while TV builds are embedded renderers. PTT has form handling, paste area, conversion trigger, and save pipeline. TV typically only displays generated tables.
