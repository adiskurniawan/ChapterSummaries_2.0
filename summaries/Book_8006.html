Conceptual Notes
Module overview markdown.py is a single-file Markdown→HTML pipeline designed to be safe, predictable, and fallback-friendly. It accepts raw markdown text and returns sanitized HTML. It prefers the external markdown library when available but falls back to a built-in renderer that implements only the needed subset of Markdown features. The module centralizes pre-processing (table and code stashing, underscore handling), conversion, and post-processing (placeholder restore, sanitization, link hardening). Inputs are plain text strings. Outputs are sanitized HTML strings. The module is intended as a drop-in replacement for small static-site or documentation renderers where predictability and safety matter more than full Markdown feature parity.
HTML sanitization (_is_safe_url, _sanitize_html) Purpose. Prevent XSS and unsafe attributes while allowing useful markup.
Behavior. _is_safe_url performs a conservative URL whitelist check. It allows http, https, mailto, relative and protocol-relative URLs. It rejects javascript: and malformed inputs. _sanitize_html escapes all angle brackets first, then rebuilds allowed tags with a small set of attributes per tag. Dangerous tags (script, iframe, embed, object, form, input, button, svg, link, meta) are stripped along with inline event handlers.
Scope. Light sanitizer; not a full HTML parser but robust for allowed tags.
Tests. XSS vectors, attribute injection, malformed HTML.
Improvement. Optionally replace with bleach if safe.
Code stashing (_stash_code, _restore_code_placeholders) Purpose. Protect fenced code blocks and inline code from being altered.
Behavior. _stash_code replaces code blocks and inline code with placeholders stored in maps. _restore_code_placeholders reinstates safe HTML using <pre><code> or <code> with language classes when present.
Edge cases. Backtick nesting, mixed fences, and very large blocks handled.
Underscore preprocessing (_preprocess_underscores_outside_placeholders) Purpose. Avoid accidental emphasis from underscores in identifiers.
Behavior. Converts escaped underscores, replaces in-word underscores with safe markers, escapes remaining underscores. Preserves placeholders registered by code or emphasis stages.
Inline cell renderer (_render_cell_inline) Purpose. Render table cell content safely using inline-only markup rules.
Behavior. Stashes inline code, replaces emphasis markers with placeholders, preprocesses underscores, escapes text, substitutes back placeholders and inline code as <code>. Result passes through _sanitize_html.
Use-case. Ensures safe inline rendering inside <td> or <th> without block-level tags.
Pipe table parser (_split_unescaped_pipes, _is_header_separator_row) Purpose. Parse GitHub-style pipe tables while ignoring escaped pipes and code spans.
Behavior. Replaces code with tokens, replaces escaped `, trims outer pipes, splits on unescaped pipes, restores code and escaped pipes.
Edge cases. Handles pipes inside code blocks.
Tests. Escaped pipe, inline code with pipes, leading/trailing pipe variants.
Table rendering (_render_table_block) Purpose. Convert parsed pipe-table lines into HTML <table> with <thead> and <tbody>.
Behavior. Normalizes header and body rows, computes alignment, renders sanitized cells via _render_cell_inline, outputs safe <th>/<td>.
Edge cases. Malformed tables return original lines to avoid broken HTML.
Table stashing (_stash_tables, _restore_table_placeholders) Purpose. Treat pipe-table blocks as pre-rendered fragments to isolate from Markdown engine.
Behavior. Stashes contiguous pipe tables, pre-renders using _render_table_block, wraps in <div class="table-container inline-mode">, replaces with placeholders. Restores placeholders at end.
Testing. Ensure stash→convert→restore preserves content.
Fallback Markdown renderer (_fallback_markdown_to_html) Purpose. Provide minimal, safe Markdown-to-HTML fallback.
Behavior. Stashes code, applies emphasis placeholders, preserves table placeholders, escapes text, restores code and table placeholders, wraps blocks in <p> split by double newlines.
Design tradeoffs. Supports small subset: emphasis, inline code, fenced blocks, pipe tables.
Tests. Validate paragraphs, code blocks, and tables.
Public API (markdown_to_html, get_markdown_renderer) Purpose. Expose stable entry points.
Behavior. markdown_to_html(md_text) normalizes text, stashes tables, converts via MarkdownRenderer with controlled extensions, sanitizes, enforces link rels. Falls back to _fallback_markdown_to_html on failure. get_markdown_renderer returns callable.
Config options. strip_table_br and nl2br flags control behavior; caching optional.
Error handling & logging Purpose. Keep conversion resilient.
Behavior. Wrap fragile operations in try/except. Logs failures, falls back safely. If both main and fallback fail, returns escaped <pre> block.
Recommendation. Set logger level for debug during development; monitor errors in production.
Security & safety layering Purpose. Combine multiple small defenses.
Behavior. Phased processing reduces attack surface: stash code/tables, neutralize underscores, controlled rendering, restore placeholders, final sanitization, link-hardening, remove <br> in cells.
Rationale. Defensive programming reduces risk.
Tests. Include malicious payloads in code, attributes, and HTML; consider fuzz tests for sanitizer.
Final notes: integration, tests, and upgrades Integration notes. Used by render.core_table, convert, core_fragments. core_table uses _render_cell_inline, convert uses markdown_to_html. Key fix: avoids auto <br> in <td>/<th>.
Test plan. Multiline cells, table roundtrips, code block preservation, underscore handling, external link rel enforcement, cache correctness.
Upgrade paths. Add configurable extensions for higher Markdown fidelity; optionally replace sanitizer with bleach; stream large inputs to reduce memory.

CLI entry (main) Scans --md-folder for *.md and *.csv. Groups files by first 9 filename chars into "book" prefixes. For each group builds all_tables and writes either renderer output or fallback HTML. CLI flags: --md-folder, --output, --no-renderer, --verbose.
Renderer loader (_load_renderer) Attempts local html_renderer.py via importlib.util.spec_from_file_location, then normal import html_renderer. Exposes render_html callable or None.
Markdown table parsing (parse_markdown_tables) Conservative pipe-style parser. Skips fenced code blocks. Uses pandas.read_csv(sep=' ', engine='python')when pandas available. Preserves inline backtick code via escape tokens. Avoids inserting<br>.
Inline markdown -> HTML (markdown_to_html) Minimal inline processing for inline ` code , bold, and italic. Stashes inline code and restores as ` with proper escaping.
DataFrame rendering (render_df_to_html_preserve_code, render_df_to_html) Replaces inline backtick code in cells with tokens, calls df.to_html(escape=True), then restores tokens as <code>escaped</code>. Uses pandas when available; falls back to minimal text rendering if not.
CSV parsing (_parse_csv_file_to_df) Tries utf-8 then latin-1. Uses pd.read_csv(..., sep=None, engine='python') then csv.Sniffer fallback with delimiters [,, ;, \t, ]. Reads as strings (dtype=str). Returns DataFrameorNone.
Fragment cleanup (_cleanup_fragments_safe) Removes .frags_* directories and .fragments files in provided target directories.
Fallback writer When renderer is absent or fails, writes simple HTML using render_df_to_html for each table.
Process flow & metrics Tracks processed_files and written_pages. Cleans fragments at end. Exit codes: 0 normal, 2 on unexpected exception.
