Ultimate Technical Enhancement Playbook for script.list.js
1. Preparation & Baseline Analysis
- Fetch Latest Script: Pull the production or repository version. Compare hashes or timestamps to ensure no mid-edit loss.
- Backup: Save as script.list.js.bak or versioned copy; confirm restore path works.
- Identify Entry Points: Document key interactive points: window.convert, __tv_do_convert, loadFilesSequentialAndHideForm, presentGroupsOnly, createGroupRowForRender.
- DOM Element Audit: Confirm required elements exist (resultsWrap, search, PASTE_ID, expandAllBtn) and log warnings if missing.
- Dependency Verification: Ensure helper functions (highlight, fetchLiveFilesSequential, loadGroupDescriptions) are loaded and operational.
- Debug & Status Hooks: Validate dbg() and updateStatus() work; wrap calls for consistent logging.
2. Functional Mapping & Feature Inventory
- UI Rendering: createGroupRowForRender, renderGroupsFromFileResults → toggles, collapsible lists, headers, file rows.
- Search & Scoring: doSearch, engine.candidatesForQuery, engine.scoreFiles → handle async, versioning, top-N results, fallback search.
- File Loading: loadFilesSequentialAndHideForm, fetchFileWithTimeout → sequential fetch, retries, timeout handling, caption injection.
- Clipboard Handling: copyTableMarkdown, copyTablePlain, getRawPasteText, fallback via textarea → cross-browser, raw/markdown toggle.
- Captions & DOM Updates: runtimeCaptions, .table-caption, safe innerHTML injection, deduplication logic.
- Event Handling: click, input, and keyboard events; debouncing; delegation; ARIA compliance.
- Accessibility & ARIA: Verify tabIndex, role, aria-expanded, aria-pressed, and labels for screen readers.
3. Risk Assessment & Edge Cases
- Async Race Conditions: Use searchVersion or similar versioning counters to prevent stale results from overwriting newer search results.
- DOM Nonexistence: Fallback if resultsWrap, paste-area, or other containers are missing.
- Failed File Fetch: Insert <!-- failed to fetch --> placeholders; continue processing remaining files.
- Clipboard Failure: Use textarea fallback if Clipboard API unsupported; show toast or console warnings.
- Caption Collisions: Deduplicate runtime captions; handle empty or conflicting names.
- Large File Sets: Limit DOM updates to top 200 items to prevent layout thrashing.
- Malformed Data: Sanitize names, captions, and content (sanitizeCaptionForInjection, safeMarkdownHeading) to prevent XSS or markup breakage.
4. Planning Enhancements
- Single-Function Focus: Apply one enhancement per cycle.
- Define Acceptance Criteria:
  - Captions correctly applied with unique IDs.
  - Search highlights match query; top matches displayed.
  - Toggle and expand/collapse functional.
  - Clipboard copy works for raw and markdown.
  - No console errors or runtime exceptions.
- Mock Data Creation: Simulate realistic file groups, captions, and query inputs for testing.
- Pre-validation: Ensure DOM structure, CSS variables, and helper functions exist.
5. Implementation Guidelines
- DOM Integrity: Maintain flex layouts, gaps, padding, border-radius, text-overflow, and ellipsis.
- Safe InnerHTML Usage: Only for sanitized content; default to textContent where possible.
- Event Handling: Delegate click/input events to persistent container elements where feasible; wrap in try/catch.
- Async Control: Maintain versioning for search; batch DOM updates; debounce inputs; schedule sequential file fetches with small delays.
- Accessibility Compliance: Preserve ARIA attributes; ensure focusable elements; keyboard navigation for toggle/load buttons.
- Error Handling: Each critical block wrapped in try/catch; failures logged via dbg() but do not halt UI.
- Clipboard Safety: Fallback for legacy browsers; respect window.__ptt_force_raw_copy.
- Code Consistency: Reuse utilities (highlight, sanitizeCaptionForInjection, safeMarkdownHeading) to ensure uniform behavior across functions.
6. Testing & Validation
- Unit Tests: Validate helper functions, sanitization, and highlighting logic.
- UI Interactions: Test expand/collapse, toggle buttons, search filtering, and paste area updates.
- Clipboard Tests: Markdown and raw copy across major browsers; test fallback via hidden textarea.
- Stress Tests: >200 file groups, long captions, rapid search inputs, simultaneous fetch requests.
- Accessibility Testing: Tab navigation, screen readers, ARIA roles, and labels.
- Error Simulation: Corrupt file URLs, missing captions, and malformed DOM to validate error handling.
7. Deployment & Monitoring
- Staging First: Validate logs, DOM updates, clipboard, toggles, search accuracy.
- Logging & Monitoring: Maintain dbg() logs for failures and timing; optionally log fetch errors to console or server.
- User Feedback Observation: Watch toggle, search, and caption application in real usage. Adjust timing or DOM updates if needed.
- Version Control: Record script version, modification date, and functional changes. Maintain changelog for auditing.
8. Future Enhancements & Maintenance
- Modularize Features: Isolate features per function for easier debugging and enhancement.
- Async Safety: Always use versioning or cancellation tokens for multi-step async flows.
- Performance Optimization: Minimize DOM operations, batch updates, limit render loops, and debounce user inputs.
- Accessibility First: Every new interactive element must support keyboard and screen readers.
- Consistent Utilities: Centralize sanitization, highlighting, and markdown heading creation to prevent inconsistency.
- Continuous Documentation: Update this table with every change; maintain explicit mapping from code to table for developer reference.
- Monitoring & Alerts: Add optional runtime warnings for unexpected DOM mutations or failed file fetches.
- Unit & Integration Tests: Keep expanding automated tests to cover newly added features or edge cases.
