Q: How do I import all XML files from a folder into Power Query?
A: Use Folder.Files("path"), filter for .xml, then parse each file with Xml.Tables([Content]) and expand resulting tables.

Steps
1. SourceFolder = Folder.Files("path").
2. XmlFiles = Table.SelectRows(SourceFolder, each Text.Lower([Extension]) = ".xml").
3. Parsed = Table.AddColumn(XmlFiles, "XmlTable", each try Xml.Tables([Content]) otherwise null).

Why it matters
Batch import automates file handling and ensures consistent schema processing.
Q: How do I safely parse XML that might be invalid?
A: Wrap Xml.Tables([Content]) in try ... otherwise so parsing failures return null (or error info) instead of stopping the query.

Pattern
1. Parsed = Table.AddColumn(XmlFiles, "XmlTable", each try Xml.Tables([Content]) otherwise null).
2. Log try error details for troubleshooting.

Why it matters
Keeps the import running and surfaces bad files for review.

Q: Why use BackgroundQuery:=False when refreshing a QueryTable?
A: It forces a synchronous refresh so VBA waits until the refresh finishes before continuing—necessary when subsequent code depends on refreshed data.

When False vs True
When False
1. Excel waits until query finishes.
2. Code execution is blocked until refresh completes.
3. Necessary when later code depends on refreshed data.

When True
1. Excel starts refresh and continues immediately.
2. VBA does not wait.
3. Downstream code may run on stale data.

Example
QueryTable.Refresh BackgroundQuery:=False vs ...:=True.

Why it matters
Avoids race conditions and ensures correct sequencing.
Q: What happens if BackgroundQuery is True and you trigger Worksheet_Change?

Use BackgroundQuery:=False or have the handler verify data readiness.

Q: How do I generate a single normalized table from many XML schemas that vary slightly?
A: Build a canonical schema (list of expected column names), parse each file to a record, extract candidate fields with a normalization function that returns the first non-empty candidate, then Table.FromRecords and Table.SelectColumns with MissingField.Ignore to enforce the canonical set.

Pattern
1. Canonical = {"NPWP","Nama","Gross",...}.
2. Normalize = (r) => [NPWP = FirstNonNull(r[CounterpartTin], r[TIN], null), ...].
3. Table.FromRecords(List.Transform(parsed, Normalize)).

Why it matters
Provides a stable output despite input schema drift.

Makes maintenance and bug fixes one-place.
