Table 3
Failure Mode / Symptom
Likely Cause & Triage Steps
Save HTML button does not work Likely cause: attachHeaderHandlersOnce did not run after header markup insertion, so #saveBtn lacks a click listener.

Triage: Verify document.getElementById('saveBtn') exists and check getEventListeners(saveBtn) in DevTools or test saveBtn.onclick. If unbound, call attachHeaderHandlersOnce() after the header is inserted.
TOC links do not exist after Convert Likely causes: buildSingleTableToc() skipped because more than one .table-wrapper exists, or #tocBar is missing, or initRenderedContent was not invoked after injecting tables into a custom area (via routeHtmlToAreas).

Triage: Confirm #tocBar is present, count .table-wrapper, and call window.PasteToTable.initRenderedContent() manually. Also inspect console for tv:extra loader diagnostics.
Convert button triggers but nothing happens Likely cause: Convert delegation not attached (rare) or convertButtonHandler errors swallowed.

Triage: Check window.__tv_check_convert() and inspect console logs. If delegation attached but existing handler throws, convert robustness patch catches and logs it and then runs fallback.
Injected HTML lands in a hidden container Cause: routeHtmlToAreas may place the HTML into #page-area and hide #tables-viewer. Save uses #page-area only if it has .visible.

Triage: Ensure your routeHtmlToAreas sets .visible when intended or that processRenderedHtml fallback path is used.
Extra.js not loaded and some enhancements missing Triage: Check console for tv:extra loader: all candidate paths failed. Use window.__tv_patch_diag() to see candidate anchors and whether extra loaded. Manually place assets/extra.js at the expected path or set window.__tv_no_auto_extra_load and manually load.

3) TV2.1 script.js reference Align naming, event delegation, and export flow to reference behavior (but keep PasteToTable-specific hooks).
Rationale: match Table Viewer v2.1 behavior while preserving PasteToTable integration.
4) index.html & selectors Ensure selectors used in script.js (IDs/classes) match templates/index.html (e.g., #tocBar, .tv-table, #copyAllTablesBtn, .table-wrapper).
Rationale: avoid selector mismatches that cause missing handlers or duplicate bindings.
5) Duplicate event bindings (issue) Identify and remove duplicate .addEventListener / .on bindings; centralize handlers using event delegation at the nearest stable ancestor (prefer document or #stickyMainHeader where appropriate).
Rationale: prevents double-firing, memory leaks, and multiple exports/copies. Use once or named handler references for safe add/remove.
6) Single source of truth for click handlers Create named handler functions attached once (e.g., handleExportClick, handleCopyClick) and reference them; avoid inline duplicate anonymous functions.
Rationale: simplifies unbinding and avoids accidental multiple registrations on re-renders.
7) Export XLSX flow & DOM triggers Ensure export reads from the canonical table DOM (.tv-table inside .table-wrapper) and does not rebind on render. Use a queued worker (if present assets/worker.js) or a one-off async function; guard export function with a small debounce and a re-entrant lock (exportInProgress).
Rationale: fixes exports that grab wrong DOM or rebind repeatedly. Debounce/lock prevents double-download.
8) Keyboard / accessibility triggers Ensure keyboard shortcuts (e.g., Ctrl/Cmd+E for export, Ctrl+C for copying current table when focused) are registered once and respect focusable elements. Make handlers conditional on document.activeElement to avoid interfering with text inputs.
Rationale: prevents accidental triggers while typing and matches accessibility expectations.
9) Collapse / toggle logic & mobile icon-only Implement collapse toggle with a single state attribute (data-collapsed="true | false") on .table-wrapper; update ARIA attributes and icon-only mobile view rules (show tooltip on long-press / hover). Ensure collapse preserves internal state on re-render without re-binding toggle handlers.
Rationale: consistent collapse state, accessible, and matches TV2.1 mobile behavior.
10) Copy-to-clipboard & persistence Centralize copy functions (copyTableAsMarkdown, copyAllTablesMd) and use the Clipboard API with fallback. Persist last-copied preference in localStorage under a namespaced key (e.g., pasteToTable.lastCopyMode). Use try/catch for permission failures and provide user feedback via existing UI elements.
Rationale: ensures consistent copy behavior and user preference persistence.
11) Prevent rebinds on dynamic render When the UI re-renders tables, use event delegation (single root listener) rather than rebinding per-table. If per-table binding is necessary, unbind first using stored references.
Rationale: prevents multiplicative bindings across multiple renders.
12) Preserve window.PasteToTable public API Detect and preserve/extend existing window.PasteToTable namespace; export minimal functions for testing (init, destroy, exportTables, copyAllTablesMd).
Rationale: keeps backward compatibility for external integrations.
13) Worker integration (assets/worker.js) Use existing worker for heavy exports when present; otherwise fallback to main-thread processing. Ensure worker is instantiated once and terminated cleanly on destroy.
Rationale: keeps performance for large tables and avoids orphaned workers.
