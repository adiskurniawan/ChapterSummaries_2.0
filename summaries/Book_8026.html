Failure Mode / Symptom
Likely Cause & Triage Steps
Save HTML button does not work Likely cause: attachHeaderHandlersOnce did not run after header markup insertion, so #saveBtn lacks a click listener.

Triage: Verify document.getElementById('saveBtn') exists and check getEventListeners(saveBtn) in DevTools or test saveBtn.onclick. If unbound, call attachHeaderHandlersOnce() after the header is inserted.
TOC links do not exist after Convert Likely causes: buildSingleTableToc() skipped because more than one .table-wrapper exists, or #tocBar is missing, or initRenderedContent was not invoked after injecting tables into a custom area (via routeHtmlToAreas).

Triage: Confirm #tocBar is present, count .table-wrapper, and call window.PasteToTable.initRenderedContent() manually. Also inspect console for tv:extra loader diagnostics.
Convert button triggers but nothing happens Likely cause: Convert delegation not attached (rare) or convertButtonHandler errors swallowed.

Triage: Check window.__tv_check_convert() and inspect console logs. If delegation attached but existing handler throws, convert robustness patch catches and logs it and then runs fallback.
Injected HTML lands in a hidden container Cause: routeHtmlToAreas may place the HTML into #page-area and hide #tables-viewer. Save uses #page-area only if it has .visible.

Triage: Ensure your routeHtmlToAreas sets .visible when intended or that processRenderedHtml fallback path is used.
Extra.js not loaded and some enhancements missing Triage: Check console for tv:extra loader: all candidate paths failed. Use window.__tv_patch_diag() to see candidate anchors and whether extra loaded. Manually place assets/extra.js at the expected path or set window.__tv_no_auto_extra_load and manually load.

Quick Diagnostic / Fix Area
Console Commands, What to Expect, and Fix Steps
Does the Save button exist and is the handler present? Inspect the element:
document.getElementById('saveBtn')

Check whether the save handler function exists:
typeof window.saveHtmlHandler → should be "function".

Check whether a click handler is bound (Chrome):
getEventListeners(document.getElementById('saveBtn'))

Expected: The saveBtn element exists and either has an event listener or saveBtn.onclick is non-null. If the element is missing or the handler is undefined/unbound, Save won’t work.
Is the header wiring function executed? Check if attachHeaderHandlersOnce ran:
!!(attachHeaderHandlersOnce && attachHeaderHandlersOnce._attached)

Expected: true. If false, header button wiring did not run (common cause of “Save not working”).
Does the TOC container exist and how many tables are present? Count wrappers and inspect TOC:
document.getElementById('tocBar')
document.querySelectorAll('.table-wrapper').length
document.getElementById('tocBar')?.querySelector('ul')

Expected: #tocBar exists and (for single-table mode) a <ul> with row anchors exists. If .table-wrapper count ≠ 1, buildSingleTableToc() intentionally skips per-row anchors.
Was the dynamic-injection re-init run after render? Check if helpers exist:
typeof window.PasteToTable?.initRenderedContent → should be "function".

If route-based insertion is used, check whether initRenderedContent was called after injection (no direct indicator unless logged). You can manually re-run:
window.PasteToTable.initRenderedContent()
Any console errors? Open console and search for errors thrown during page load or after Convert. Errors can stop attachHeaderHandlersOnce or initRenderedContent from completing. Fix errors first.
Immediate fix: force re-bind header handlers If attachHeaderHandlersOnce._attached is false or saveBtn has no listener, run:
attachHeaderHandlersOnce()

Then re-check getEventListeners(saveBtn) or click Save to test.
If save handler exists but Save downloads nothing Ensure containers contain expected markup:
document.getElementById('page-area')?.classList.contains('visible')
document.getElementById('tables-viewer')?.innerHTML.length

If #page-area has content but not .visible, toggle manually to test or ensure #tables-viewer contains content.
Re-run re-init after dynamic injection Safe test:
window.PasteToTable && window.PasteToTable.initRenderedContent && window.PasteToTable.initRenderedContent()

After running it, check #tocBar for anchors and saveBtn for binding.
If TOC per-row anchors missing but one table exists Force TOC generation:
buildSingleTableToc()

If anchors appear, the issue was timing. Long-term fix: ensure initRenderedContent runs after injection.
Check for name/id mismatches If your template renamed IDs, restore expected ones (saveBtn, tocBar) or adapt integration. Quick check in Elements panel: confirm IDs match script expectations.
Persistent Fix A – Ensure header handlers attach after render Problem: attachHeaderHandlersOnce() runs before header markup exists.
Fix: Call it after header injection or ensure script loads after header HTML. Inline onclick as fallback (least preferred).
Why: saveHtmlHandler is defined but never bound otherwise.
Persistent Fix B – Ensure initRenderedContent executes after injection Problem: Dynamic insertion places tables but doesn’t call initRenderedContent().
Fix: Call it after DOM insertion or update routeHtmlToAreas to do so.
Why: TOC & button wiring happen inside initRenderedContent.
Persistent Fix C – Per-row TOC in multi-table pages Problem: buildSingleTableToc() skips multi-table.
Fix: Use table-level TOC (recommended) or implement custom multi-table row TOC builder.
Persistent Fix D – #page-area visibility semantics Problem: Save uses #page-area only when it has .visible.
Fix: Ensure .visible is set or place content in #tables-viewer.
Tests after fixes Save: Click Save, confirm tables.html downloads with table HTML.
TOC: Check anchors (single-table = per-row, multi-table = table links).
Convert: Paste & Convert, inspect console for errors, re-run initRenderedContent() if needed.
If still failing — collect diagnostics Copy console errors. Run:
typeof window.saveHtmlHandler
typeof attachHeaderHandlersOnce
window.PasteToTable.initRenderedContent()
Observe behavior for clues.
Quick remediation decision tree - If saveBtn missing → fix header ID.
- If saveBtn present but unbound → call attachHeaderHandlersOnce() after header insertion.
- If #tocBar empty and one table → call buildSingleTableToc() or run initRenderedContent().
- If #page-area has content but Save fails → mark .visible or use #tables-viewer.
Prevention & monitoring - Add console.info in attachHeaderHandlersOnce temporarily to confirm execution.
- Expose a debug button in dev that calls:
attachHeaderHandlersOnce(); window.PasteToTable.initRenderedContent(); buildSingleTableToc(); updateRowCounts();
- Optional automated test: trigger Convert, assert Save works and TOC anchors exist.
