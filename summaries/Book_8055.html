Table 1
Enhancement
Goal
Store human-readable descriptions per group of tables, not per individual table. Provide Name and a Description listing for all groups.
Filename & Location
groups_index.json in project root. One file contains all groups.
Primary Schema
{ "version":1, "generated_at":"<iso>", "groups":[ { "name":"Book_8051", "description":"First book summaries", "files":["Book_8051_01.md","Book_8051_02.md"] }, ... ] }
Required fields per group
name (group name), description (string), files (array of filenames belonging to this group).
Optional fields per group
url (link to first file or representative page), tags (array), count (number of files).
Character limits
name ≤ 50 chars. description ≤ 200 chars.
Encoding & format
UTF-8 JSON. Compact. \n line endings.
Atomic updates
Write to temp groups_index.json.tmp then os.replace() to final path.
Generation strategy
Background task or on-save hook: scan STORAGE_PATH, group files by prefix or folder, create group entries with descriptions, write JSON.
Trigger points
1) After batch save.
2) Server start.
3) Manual admin trigger.
4) Periodic cron.
Consistency model
Eventual consistency acceptable. Document lag window (~10–30s).
Conflict handling
Atomic replace prevents corruption. If multiple updates race, regenerate fully from current storage state.
Access pattern (server)
Endpoint /groups/index returns JSON with all groups and descriptions.
Access pattern (client)
Client reads JSON and displays a Name and Description table with optional search/filter bar.
Fallback behavior
If JSON missing, fallback to scanning folder prefixes and list group names without descriptions.
Validation rules
Validate JSON schema, ensure no absolute paths, reject malformed writes and log failures.
Size limits & pagination
Keep file size reasonable (<1MB typical). If many groups support client-side paging.
Security
Readable by server only. Escape descriptions to prevent XSS.
Performance
Serving static JSON is fast. Cacheable with Cache-Control: max-age=10.
Versioning
version field to handle schema changes.
Backups & rollback
Keep last N copies (e.g. groups_index.json.1) for recovery.
Extraction heuristics
Description per group can be the first line of the first file or manually defined in JSON.
Normalization rules
Collapse whitespace, trim length, remove control characters.
Admin override
Allow manual override per group in JSON.
Search indexing
Optionally precompute a normalized search field per group for faster search/filter.
Filtering & query params
Server /groups/index can support ?q=, ?prefix=, ?tags= filters.
Audit logging
Log updates: timestamp, initiator, number of groups, duration.
Testing checklist
1. Create sample groups.
2. Generate JSON.
3. Load client table.
4. Search/filter works.
5. Atomic replace succeeds.
Operational concerns
On network filesystems allow for eventual consistency. Retry if write fails.
Maintenance
Provide CLI tool for offline regeneration and validation.
Safety & compliance
Truncate and sanitize descriptions to avoid leaking sensitive content.
Recommended defaults
Auto-generate after batch saves (async). Full regeneration every 10–15 minutes. Description target ~100–150 chars.

Table 2
Enhancement
Group metadata editor
Allow admins to edit group name, description, tags, and url from a compact UI. Changes write to groups_index.json with atomic replace.
Tagging system
Add lightweight tags on groups and files. Client can add/remove tags. Server stores tags in groups_index.json and supports ?tags= filter.
Quick-export ZIP
One-click export of all files for a prefix into a ZIP named <prefix>_export.zip. Streamed response to avoid large memory use.
Webhook on batch save
Optional webhook URL per instance. POST summary (prefix, files, status) after batch save finishes.
Scheduled auto-export
Periodic job that zips recent groups and places archive in exports/ or emails link. Configurable interval.
Auto README per group
Generate Book_8052_README.md summarizing group files and first-line descriptions. Update on batch save.
Simple versioning
Keep timestamped copies like Book_8052_01.20251028.md when enabled. Index versions in groups_index.json.
Restore UI
Allow selecting and restoring a previous version to active filename. Do atomic replace and log action.
Preview pane
Client shows rendered markdown preview side-by-side before save. Small JS renderer only.
Bulk rename / re-prefix
Client can rename a group prefix; server atomically renames matching files (with collision checks) and updates index.
Duplicate detection
Detect identical content by SHA256 and mark duplicates in index. Offer dedupe action.
Lightweight ACL flags
Per-group flags: public, read-only. Enforced server-side; reflected in groups_index.json.
Search-as-you-type
Client provides instant search across name, description, and tags. Server can supply normalized index for performance.
Chunked client upload
Client uploads very large batches in chunks with resume support. Server assembles and writes atomically.
Mobile-first save action
Mobile-optimized single-button save with icon-only UI and optional prefix suggestion from clipboard.
Minimal telemetry page
Internal page showing counts: groups, files, last save time, error rate. No PII; useful for ops.
