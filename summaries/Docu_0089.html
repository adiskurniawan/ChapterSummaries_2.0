Technical Breakdown
assets/script.toc.js
Role/Purpose
TOC builder for PasteToTable. Builds anchor links for row headings or table headings. Supports single-table auto “Topic N” labeling. Avoids CSS injection. Robust heading normalization.
Boot/Config Flow
Defines global PTToc object once. Registers init on DOM ready. Loads config defaults. Marks install flag to prevent double load.
Logging
No explicit logging. Silent error catches across functions.
Storage Preparation
No persistent storage. Uses in-memory flags (__pt_toc_installed, _mutationObserver, timers).
Asset Preparation
Dynamically creates list elements for TOC. No CSS injection. Uses provided selectors.
Flask App Creation
Not applicable.
Request & Upload Safety
No network. DOM-only operations. Sanitizes text. Normalizes headings to avoid line break artifacts.
Optional Integrations
Config accepts overrides. Works with sticky header anchor offset. Can coexist with custom CSS.
CLI Surface
None. Access via window.PTToc and .init().
Error Handling
Heavy try/catch. Fails closed. Swallows exceptions to avoid breaking page. Fallbacks for DOM queries.
Observability
No telemetry. Silent mutation observer rebuilds.
Signals & Graceful Shutdown
Disconnects previous MutationObserver if exists. One delegated click handler.
Security Best Practices
No HTML injection. Text labels set via textContent. ID generation strips invalid chars. Minimizes attack surface.
Testing & Import Safety
Self-executing closure. Guards against double init. Works if script loaded late or early.
Performance & Resource Guidance
Debounced rebuild (150ms). MutationObserver for subtree. Minimal DOM churn: clears and rebuilds lists. Efficient ID creation.
Operational Recommendations
Override selectors via init config. Ensure unique sticky header ID if using offset. Provide CSS externally.
Conceptual Skeleton
Init → select TOC roots → detect tables or headings → generate stable IDs → build list items → attach click delegation → highlight active → observe DOM changes → debounce rebuild.
Risks & Hardening Notes
Silent catch hides failure origin. Potential performance cost on heavy DOM churn. ID collisions resolved via suffix. No ARIA for list container; links get role and aria-label.
Final Summary
Stable lightweight TOC generator. Uses robust heading text normalization. Auto row anchors for single-table mode. Mutation-aware and debounced. Defensive coding with silent fallbacks.
