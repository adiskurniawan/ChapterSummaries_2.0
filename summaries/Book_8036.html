Table 2
Logic / Structure
Conceptual Notes
Identify Render button Detect #renderBtn, fallback to #addBtn, #render, #convertBtn only if render not found. Do not assume #convertBtn is the render control.
Change text and handler Change visible text to Add Table. Wrap existing convert/render invocation rather than replace it. New handler calls existing convert function to produce HTML then append new table HTML to container. Keep original convert flow intact.
Preserve multiple tables Remove any code path that clears the container on render. Append inserted .table-wrapper nodes. Ensure new wrappers get unique data-table-id and id attributes (incremental index).
Initialize per-table functions After appending table HTML run the same initialization steps used in initRenderedContent for that single inserted wrapper: ensure .table-container exists, call _ensureRowUidsAndSnapshot, set up sortStates, store data.origHtml on cells, attach per-wrapper handlers (toggle, copy, export, sort), call optimizeTableControls, update row counts. Prefer reusing existing functions rather than duplicating logic.
TOC updates If window.PTToc present call PTToc.build() for incremental update. Otherwise call buildTocFromCaptions() and buildSingleTableToc() for the new table. Use short setTimeout calls to let DOM settle.
Source snapshot & copy behavior Ensure window.__tv_last_source is captured before convert (existing snapshot logic present). When appending table, attach data-source-snippet or rely on existing getPreferredSourceForTable to prefer window.__tv_last_source.
Preserve global features Do not change back-to-top, search, export-all, or keyboard handlers. Ensure appended tables are included when exportAllBtnHandler, searchTable, copyAllTables*, and other global handlers iterate .table-wrapper.
Idempotence & safety Handler attachment must be idempotent (use dataset flags). Do not rebind global event listeners. Use try/catch to avoid breaking the page if functions absent.
Accessibility & attributes Ensure new wrappers include same attributes (data-table-id, data-table-alignments, id on heading) to preserve ARIA and TOC anchors.
Fallbacks & compatibility If convert returns raw HTML string with multiple wrappers, append each; if it returns non-wrapper HTML, wrap it with .table-wrapper preserving tools if possible. Maintain parity with Table Viewer 2.1 DOM structure.
Minimal file touches Primary edit: assets/script.js (or assets/script.table.js depending where render binding lives). No modifications to render/convert.py unless a compatibility issue is found later.
