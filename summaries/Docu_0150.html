Business Continuity Plan (BCP) — Payroll Processing
1. Purpose
Ensure payroll can be calculated, approved, and disbursed accurately and on time during disruptions (system failure, data loss, office closure, staffing shortage, or vendor outage).
2. Scope
Covers all activities from data collection to salary disbursement:
• Attendance and variable data extraction
• Payroll calculation
• Review and approval
• Bank file creation and transfer
• Statutory contributions and tax submissions
• Payslip distribution
• Payroll record archiving
3. Critical Payroll Inputs
• Employee master data
• Attendance/leave/overtime
• Allowances, deductions, benefits
• Bank account details
• Tax information (e.g., PTKP, progressive brackets)
• Statutory rates (BPJS, pension, etc.)
4. Roles and Responsibilities
• Payroll Officer: Data preparation, calculation, file generation
• Payroll Supervisor: Validation, exception handling
• HR Manager: Final approval, communication
• Finance/Treasury: Bank upload and payment release
• IT Support: System restoration, backup retrieval
5. Key Risk Scenarios and Actions
5.1 Payroll System Downtime — Response
• Activate backup payroll system or local spreadsheet template.
• Retrieve latest payroll backup file.
• Perform manual calculation for essential components (basic pay, fixed allowances, statutory deductions).
• Prioritize payment of fixed salaries; process variables later if necessary.
• Maintain audit log of all manual entries.
5.2 Data Loss or Corruption — Response
• Restore payroll database from last available backup.
• Validate restored records (master data and variables).
• Re-enter missing data based on attendance system reports, HR records, and prior month references.
5.3 Staff Unavailability — Response
• Use trained backup personnel.
• Access payroll procedure manual and work instructions.
• Grant temporary payroll system access rights to replacement staff.
5.4 Office Closure (Disaster, Pandemic, etc.) — Response
• Activate work-from-home protocol.
• Use VPN/secure remote access for payroll system and bank portal.
• Enable digital approvals via approved channels (DocuSign, internal workflow).
5.5 Bank Portal Unavailable — Response
• Use secondary bank account or secondary bank portal if available.
• Issue manual transfers for critical employees (management-approved).
• Provide delayed payment communication if required.
6. Backup and Recovery Measures
• Daily automated system backup; weekly full backup.
• Offsite backup storage (cloud or DR site).
• RTO target: Restore within 4 hours.
• RPO target: Data loss not exceeding 24 hours.
7. Manual Payroll Procedure (Fallback Mode)
• Use locked Excel payroll template with formulas embedded.
• Import attendance data manually from CSV/exports.
• Calculate gross, deductions, and net pay.
• Generate payment list for Finance.
• Generate manual payslips if needed (PDF/Excel).
8. Communication Plan
• Notify HR, Finance, IT, and Management within 30 minutes of disruption.
• Update employees only if payout timing is affected.
• Provide status updates until payroll is completed.
9. Compliance and Audit
• Maintain logs of all deviations from standard payroll process.
• Store manual calculation sheets for 7 years (or per company policy).
• Ensure statutory submissions remain on schedule.
10. Review and Testing
• Annual BCP simulation test with HR, Payroll, Finance, IT.
• Post-incident review after every disruption.
• Update the plan as organization or regulations change.
