Required Fix / Implementation Notes
1. Active link tracking
 Uses bounding-box midpoint comparison inside scroll handler. Sometimes misfires when two tables are near each other (e.g., Table 1 & Table 2 on Book 8022). Uses IntersectionObserver with dual threshold (0.4–0.6). Only marks the table visible in the center of viewport as active. Extremely stable and efficient. Replace manual scroll math with IntersectionObserver. Fallback to scroll handler only if IO unsupported. Use thresholds: rootMargin: "-20% 0px -60% 0px".
2. Smooth scroll offset
 Hard-coded offset = 80 px; doesn’t adapt if user resizes or uses custom header height. Reads sticky header height dynamically via document.querySelector('.header, #header')?.offsetHeight. Guarantees top of table caption is visible. Compute offset dynamically at scroll time. Store latest height in variable for reuse.
3. ARIA sync / accessibility
 Adds aria-current="page" on active item, but older ones sometimes remain set due to timing. Clears all aria-current attributes before applying new one. Updates in sync with active class for keyboard and screen reader coherence. Add helper clearAriaStates() to remove aria-current from all TOC links each cycle before setting new active.
4. MutationObserver scope
 Observes the entire <body>; fires on many unrelated changes, wasting cycles. Observes only config.contentSelector, typically #content or .chapter-body. Faster and safer. Change observer target to resolved content container. Disconnect during rebuilds to prevent loops.
5. Anchor hydration / duplicate prevention
 Checks if <li> for ID exists; re-adds if not. Can double-bind click handlers when tables reloaded dynamically. Uses a data-toc-bound="1" attribute on each link to mark bound anchors; never re-binds. Also detects pre-existing static TOC entries and augments them with missing metadata. Add safe binding guard → skip links already having data-toc-bound.
6. Keyboard navigation
 Mouse-only. No response to Enter/Space when focused. Supports keyboard: pressing Enter on a focused TOC link triggers smooth scroll to target. Add keydown listener on links; if key is Enter (13) → simulate click handler.
7. Performance on large documents
 ~250–300 ms initialization on 100+ tables (too many direct DOM insertions). < 200 ms target. Builds all <li> entries in a DocumentFragment, then appends once. Build TOC items into fragment; append at end for one paint cycle.
8. ARIA list roles
 Applies role="list" only on UL but not on each LI. Applies both role="list" on UL and role="listitem" on LI for perfect screen-reader mapping. Add li.setAttribute("role","listitem") during creation.
9. Scroll event cleanup
 Adds listener but never removes on teardown. Cleans up before re-init or when page navigation occurs. Expose a destroy() method to remove scroll + observer + listeners.
10. Integration with static anchors (#Table1)
 Works only if generated dynamically. TV 2.1 works for both static pre-built TOC and dynamic injection. On init, hydrate any existing links by matching IDs in DOM; skip re-creation.
11. Scroll highlight precision
 Updates immediately on scroll; flickers near viewport edges. Uses requestAnimationFrame throttling to 60 FPS max. Wrap scroll logic in rAF loop to avoid flicker.
12. CSS class sync
 Uses pt-toc-active but sometimes remains on multiple links. Ensures exactly one link has pt-toc-active. Synchronize with ARIA state update.
13. Dynamic table additions
 Observer rebuilds entire TOC whenever any .table-caption is added. Adds only the new entries incrementally. In observer callback, process mutation.addedNodes selectively.
14. Mobile behavior (TOC icon-only mode)
 Not integrated with mobile toggle. TV 2.1 collapses to icon; clicking re-opens overlay TOC. Ensure init() gracefully handles hidden TOC element; skip if none visible.
15. Namespacing & API exposure
 Exposes window.PasteToTable.toc partially. Fully namespaced: window.PasteToTable.toc.init(), .destroy(), .refresh(). Finalize clean namespace API for consistency.

16. Lazy initialization
 TOC builds immediately on DOMContentLoaded, may run before large tables fully rendered. Defers TOC creation until first animation frame after DOM load (ensuring all .table-caption nodes exist). Wrap init() in requestAnimationFrame or small setTimeout(0) for safety.
17. CSS variable awareness
 Hard-coded offset and colors (no CSS var reads). Reads computed style (like --header-offset) when available for better theme sync. Use getComputedStyle(document.documentElement).getPropertyValue('--header-offset') as dynamic offset if defined.
18. Memory & event safety
 Global listeners persist even after re-render; can duplicate on SPA reload. Cleans up listeners and observer when destroy() is called. Implement destroy() and reuse init() safely for single-page or AJAX refreshes.
19. State persistence
 Scroll position resets if TOC is rebuilt. Preserves last active ID; restores highlight after rebuild (for MutationObserver updates). Store activeId and reapply after DOM mutation rebuild.
20. Debug trace & telemetry (optional)
 No internal logs. Uses if (window.PTT_DEBUG) conditional console messages for dev builds only. Add guarded debug logs that don’t affect production performance.
