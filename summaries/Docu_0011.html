Feature
Tables Viewer v2.1
PasteToTable
Purpose
 Browser-based interface for displaying, filtering, and manipulating structured tabular data with real-time updates. Browser-based utility that converts raw text (CSV, TSV, or Excel clipboard data) into a clean, interactive HTML table for further manipulation or export.
Core Components
 - UI Layer: HTML table container with virtual scrolling; search and filter controls; column sorting and visibility toggling.
- Data Engine: Loads JSON, CSV, or API data; supports dynamic data binding; event-driven change propagation.
- Renderer: Efficient DOM patching; lazy render off-screen rows.
- Persistence: LocalStorage for user settings; optional cloud sync API. 		- Input Processor: Parses clipboard text into structured arrays; auto-detects delimiters; trims whitespace; normalizes line endings.
- Table Generator: Builds <table> dynamically; alternating row styles; optional header detection.
- Editor Tools: Add/remove rows/columns; inline cell editing with validation; copy/export to CSV or JSON.
- Integration Hooks: Can send parsed data to Tables Viewer v2.1; extensible via core_compat.py.
Modules
 - script.js: Main control layer.
- data_handler.js: Parses, validates, and transforms data.
- ui_controls.js: Manages user interaction and events.
- render_engine.js: Handles DOM rendering and visual updates.
- storage.js: Manages persistence and config save/load. 		- paste_parser.js: Main parsing logic.
- table_editor.js: Manages editable UI.
- export_tools.js: Handles CSV/JSON export.
- core_compat.py: Cross-project compatibility layer for data exchange.
Performance Targets
 - Render up to 100k rows smoothly.
- Sub-50ms updates on user interactions.
- Memory footprint <150MB for full dataset in browser. 		- Handle pasted data up to 10k rows instantly.
- <100ms parsing time for 1MB input.
- Zero reflows beyond viewport.
Security
 - Sanitization of all data before DOM insertion.
- No external eval or dynamic script injection.
- CSP compliance. 		- Input sanitization for clipboard content.
- No HTML injection from pasted text.
- CSP and sandboxed iframe compliance.
Shared Development Rules
 - Coding Style: ES6+ for JavaScript modules; Python 3.11+ for compatibility layer; modular, functional design.
- Naming Conventions: camelCase for JS variables/functions; snake_case for Python; UpperCamelCase for class definitions.
- Versioning: Semantic versioning (major.minor.patch); each release documented with changelog.
- Documentation: Inline JSDoc for functions; Markdown-based project docs; diagrams for module relationships.
- Testing: Unit tests for all parsers, renderers, and data transformers; manual UI regression checklist.
- Extensibility: Each module exposes a minimal API; shared event bus for cross-module communication; interoperable via core_compat.py. 		- Same as Tables Viewer v2.1.
Goal Alignment
 Focuses on presentation and interaction. Focuses on data ingestion and transformation.
Both converge toward a unified ecosystem for browser-based table management.

Observations / Key Differences
Purpose / Goal
 Converts pasted markdown or CSV/TSV clipboard data into clean, interactive HTML tables for export. Browser-based interface for viewing, filtering, and manipulating large tabular datasets with virtual scrolling. PasteToTable focuses on raw text ingestion + table generation; Tables Viewer focuses on large dataset display and interactivity.
Header / Sticky Bar
 <div id="stickyMainHeader"> contains title, buttons, search, and paste toggle. Fully accessible, preserves your current layout. <div id="stickyMainHeader"><div id="tv-header">…</div></div> contains search, theme toggle, and table action buttons; less complex UI, minimal DOM. Both use sticky headers; your PasteToTable header is more complex and already approved. No change should be made here.
Search
 <input id="search"> within header row, integrated with script.js. <input id="searchBox" class="tv-search"> in header. Functionally similar; your current input supports PasteToTable workflow.
Buttons / Controls
 Export, Clear, Toggle Paste, Convert, Render, Save HTML — linked to script.js. Copy, Export CSV/JSON/XLSX/PDF/Markdown, Collapse, Theme Toggle — inline JS. Your setup uses script.js for all actions; Tables Viewer has inline per-table buttons. Delegation handled differently.
Paste Panel
 <div id="paste-panel"> with form and <textarea> for input; toggleable visibility. Not present (Tables Viewer expects data already loaded). Your page supports progressive enhancement and non-JS fallback; TV v2.1 is JS-only.
Tables / Rendering
 <section id="tables" class="table-container"> dynamically populated; normalizes tables, adds two-col where needed. <div class="table-wrapper"> for each table with .table-header-wrapper + copy/export buttons; includes table sorting <button class="sort-btn">. PasteToTable currently does not implement per-column sorting in HTML; TV v2.1 has explicit sortable headers with icons.
Scripts
 script.js deferred; optional paste_handler.js/table_manager.js; mutation observers for toggles; table normalization helper; service worker optional. Inline scripts for event delegation; opacity fade-in; DOMContentLoaded for delegation; all tables immediately rendered. TV v2.1 has more granular per-table buttons and sorting; your script.js handles global table actions.
Virtualization / Performance
 Targets up to 10k pasted rows; normalizes DOM; avoids reflows outside viewport. Targets 100k rows; uses opacity transitions + lazy render; per-table DOM virtualization. TV v2.1 optimized for large datasets; PasteToTable optimized for moderate clipboard inputs.
Accessibility / ARIA
 Roles defined: banner, toolbar, main, contentinfo, aria-live, aria-controls. Minimal ARIA; uses role="region" for tables viewer, label for TOC. Your page is stronger on accessibility; TV v2.1 is functional but minimal.
Export / Clipboard
 Global Export All; Save HTML; clipboard integration via script.js. Per-table copy/export buttons; delegation logic for hidden buttons. TV v2.1 handles multiple export formats per table; PasteToTable has simpler global approach.
TOC
