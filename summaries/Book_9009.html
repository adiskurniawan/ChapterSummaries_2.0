Topic
Summary
Overview Compact, repeatable, safe workflow for conservative single-file edits and controlled multi-file refactors.
Principles Safety first; silent absorption; approve-before-next; reversible patches; test-first; in-session delivery; traceability; minimal commits.
1. Intake User supplies project tree and target files or "all". Select mode: Analysis / Conceptual Expansion / Upgrade / Audit. Provide constraints (single-file-only, allowed packages, CI access).
2. Preflight analysis Read repo tree and listed files. List missing files. Run static checks (syntax, imports, signatures). Risk scan for I/O, network, subprocesses, native code, security-sensitive areas.
3. File-by-file loop For each file (max 50 per batch): silently absorb; produce 2-col conceptual table; list risks and tests; request approval; on approval deliver patch, tests, 10-point checklist, rollback instructions and checksum; wait for test confirmation before next file.
Patch deliverables Full patched file content; focused unit tests or sample run commands; single-file diffs; example inputs/expected outputs; rollback commands; original checksum.
4. Consolidation Produce consolidated conceptual map; suggested incremental upgrade roadmap; optional larger refactors with risk/benefit analysis.
5. QA & Release Pre-merge checklist (linters, unit tests, security scan, perf smoke tests). Provide CI snippet (GitHub Actions) and artifact packaging guidance.
6. Post-deploy Monitoring suggestions and rollback playbook. Optional small follow-up audit after user runs logs/tests.
Tools & Artifacts Recommend pytest, flake8/stylelint, GitHub Actions templates. Deliver changelog entry, reason-for-change note, and "how to review" guide.
Security & Privacy Do not store secrets or execute provided code. Warn about external dependencies and network calls. Provide sanitizer-safe HTML/JS suggestions.
Operational Rules No asynchronous work. All results produced in-session. Always ask approval before next file change. On "remember/save" store as Prompt_07 and reference later.

Enhancements: HTML/React dashboard showing pending files, test status, coverage %, dependency audit, verification checklist; auto-refresh.
Benefit: reduces cognitive overhead; speeds audits.

Enhancements: run each file patch in lightweight container; preload deps; auto-clean containers.
Benefit: removes “works on my machine” issues; ensures CI parity.

Enhancements: optional multi-OS runners (Linux, Windows, macOS); check Python/JS versions; auto-report env-specific issues.
Benefit: enterprise-level QA confidence.

Enhancements: AI flags optimizations and security concerns in dashboard; analysis-only by default; no auto-mods without approval.
Benefit: speeds human review while keeping control.

Enhancements: expand checklist to 20 points (security scan, coverage≥80%, dependency audit, UI diff, accessibility, theme checks); auto-fill where possible; require manual confirmation for subjective items.
Benefit: fewer oversights; more repeatable QA.

Enhancements: auto-update changelog from patch metadata; auto-generate per-file summaries and verification results; output markdown/HTML artifacts.
Benefit: transparent and traceable history.

Enhancements: natural-language diff summaries (e.g., added functions, modified CSS rules); display alongside checklist in dashboard.
Benefit: speeds review and decision-making.
