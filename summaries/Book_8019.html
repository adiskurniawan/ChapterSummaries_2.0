Table 10
Step
Logic / Structure
Conceptual Notes
1 assets/script.js
Each file in assets/js/ (core_init.js, table_ops.js, export_xlsx.js, etc.) is now standalone, namespaced, and verified. 		Each module handles a single concern — initialization, table operations, exports, state, TOC, UI, and compatibility.
2 script.js becomes the orchestrator — a lightweight entry script that loads and coordinates all submodules. It ensures startup sequence and backward compatibility with old TV2.1 global expectations.
3 script.js contains only the root boot logic, safety guards, and dependency ordering. No UI or feature code stays here. All logic resides in assets/js/.
4 The file should initialize once (if (window.__ptt_initialized) return;) and call PTT.init() or similar. Prevents double execution when used with service workers or dynamic reloads.
5 Output size target: < 10 KB minified (old one was ~60–70 KB). Smaller, faster, safer to cache and update.
6 Build method: concatenate in strict order → compress → verify top guard and namespace. Order: compat → utils → core_init → toc_manager → state_persist → table_ops → ui_feedback → export_csv → export_xlsx.
7 Final content: no inlined event handlers, no window.onload duplicates, no DOM blocking. Clean async load ensures resilience and mobile readiness.
8 Role: act like a “glue file,” not an application script. Makes updates modular — future changes happen only in submodules.
