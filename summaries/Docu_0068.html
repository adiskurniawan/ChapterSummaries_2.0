Proposed Fix / Recommendation with Details
1. Expand all / Collapse all button updates incorrectly after search – Currently, the code determines whether any group is collapsed using groupControls.some(g => g.toggleBtn.getAttribute('aria-expanded') === 'false'). After a search, groupControls is regenerated, so this check can produce a misleading result. Maintain a dedicated global boolean allExpanded that tracks the intended state of the expand/collapse action. Initialize it as false (collapsed). Whenever rows are rendered or re-rendered (including after search), explicitly set each row’s aria-expanded and toggle button text based on allExpanded. This avoids relying on DOM inspection which may be inconsistent after DOM regeneration.
2. Button label flips incorrectly after search – The button text may still show “Expand all” while new rows are already expanded. After renderGroupsFromFileResults completes, immediately call a dedicated updateExpandButtonLabel() function. This function reads allExpanded and updates the button text and aria-pressed accordingly, ensuring the label always reflects the visual state of rows, even if they are recreated during search.
3. Toggle buttons per group not synced with global button – Newly rendered rows default to collapsed (fileList.style.display='none'), but the global button may still indicate “Collapse all”. Iterate over groupControls right after rendering each search result. For each row, explicitly set toggleBtn.setAttribute('aria-expanded', allExpanded ? 'true' : 'false') and fileList.style.display = allExpanded ? 'flex' : 'none'. This ensures every row reflects the current global expand/collapse state.
4. Debounce / async render mismatch – The doSearch function is debounced and uses async engine.scoreFiles. The global expand button may update before the new rows are fully added. Move the call to update expandAllBtn inside the .then() of scoreFiles after DOM insertion. This ensures button label and state are updated after all new rows exist in the DOM.
5. Separate per-row expand state vs global state – Currently, per-row state relies only on DOM attributes (aria-expanded). After search, rows are recreated, losing previous expand state. Maintain a property row.expanded = true/false on each rowWrap. When toggling a row, update this property. When rendering new rows after search, default row.expanded = allExpanded and apply display accordingly. This makes individual toggles persistent relative to the global state.
6. Race conditions during rapid search typing – Multiple async searches in flight can lead to outdated results rendering last, breaking button state. Implement a simple render versioning system: increment a searchVersion counter at each input. When the async scoreFiles promise resolves, only apply results if its version matches the latest searchVersion. This prevents stale DOM updates from changing expand button or group rows incorrectly.
7. Edge case: zero search results – When no matching groups exist, the global expand button may remain visible or labeled incorrectly. After rendering zero results, hide or disable the expand/collapse button, and ensure allExpanded is reset to false. This prevents user confusion.
8. Accessibility / ARIA compliance – Current code sets aria-expanded but does not sync aria-pressed reliably with visual state. Ensure that both per-row toggleBtn aria-expanded and global expandAllBtn aria-pressed are updated whenever rows or state changes. Also update screen-reader text if needed.

Step & Description
Technical Details / Notes
1. Track per-row state: Each group row keeps an object { expanded: boolean, visible: boolean, fileElements: [...] } to avoid rebuilding DOM on toggles. Maintains current expand/collapse state and visibility efficiently.
2. Keep all rows in memory: On initial render, create all group rows once and store them in an array, using display:none for hidden ones instead of removing from DOM. Reduces DOM creation and destruction overhead.
3. Search updates visibility only: Filter rows by search results and toggle rowWrap.style.display = visible ? 'flex' : 'none' without recreating rows. Makes search instantaneous and prevents losing expand/collapse states.
4. File list elements are persistent: Each fileList is created once. Toggling just changes fileList.style.display. Avoids recreating <div>s for files and keeps event listeners intact.
5. Top-N per group logic: Optionally hide lower-ranked files by toggling fileRow.style.display instead of rebuilding the list. Maintains performance for groups with many files.
6. Global expand/collapse: Iterate over groupControls and set row.expanded and fileList.style.display based on allExpanded. Always reflects the true overall state correctly.
7. Debounced search: When search value changes, compute matching group indices and toggle rowWrap.style.display for each row. Update expand button based on filtered set. Keeps UI responsive even for large datasets.
8. Highlight tokens dynamically: Update innerHTML of name/caption lines to highlight matched tokens, without creating new rows. Provides visual feedback without DOM thrashing.
9. Fallback for large groups: Optionally use virtual scrolling inside fileList for very large groups to keep DOM light. Ensures smooth scrolling and rendering performance.
