_wrap_each_table_with_ui()
 In PTT version, numbering cells appear shifted, especially rightmost column misaligned. The wrapper reconstructs <table> containers to inject toolbar, metadata, and possibly extra <div> layers. When done without fixed width or proper <colgroup>, browsers auto-balance widths based on first visible column content. Inside _wrap_each_table_with_ui, add explicit <colgroup> for each table with column count detection. Example:
cols = html.count('<td') // rows → insert <colgroup><col style="width:auto;">…</colgroup> preserving count and balance.
Missing right alignment rule
 Numeric columns should right-align (like TV) but PTT defaults to left. TV CSS defines td:last-child{text-align:right;}; PTT’s injected CSS omits it. Confirm CSS injection in _inject_ui_assets includes .table-ui-wrapper table td:last-child{text-align:right;}.
HTML ordering drift
 Numbering cell index position may flip when render_table() loops through DataFrame columns. render_table() probably reads df.columns in default order; TV may reorder or pin numeric index last. Inspect _df_to_html_preserve_code() or equivalent DataFrame→HTML conversion. If index numeric, reinsert it as last column using:
df = df.reset_index().reindex(columns=[*df.columns[1:], df.columns[0]]).
Style cascade loss
 PTT version injects table_ui.css late in <body> rather than <head>. Browser may render early incomplete layout. _inject_ui_assets() appends to body if <head> not found early. Force <head> insertion first. Modify search order: find <head> tag first; only fallback to <body>.
Minification side effect
 Minifier collapses whitespace and newlines between <td> tags, which in some engines affects inline-block width on numeric-only cells. _maybe_minify_html() not filtering <td>-dense sections. Add skip_regions=['<table', '</table>'] to minifier regex to preserve internal spacing.
Caption interference
 PTT wraps table captions inside <div class="table-caption"> causing right column to reflow incorrectly. Block display breaks table’s internal width calculation. Change to <caption> tag within <table> rather than outside wrapper div.
CSS reset variance
 TV applies border-collapse:collapse; table-layout:fixed;. PTT injects only border-collapse. Missing table-layout:fixed makes browser calculate widths per content. Enforce table-layout:fixed;width:100%; in injected CSS.

Technical Analysis
Root Cause Evidence
Corrective Action (Verified)
1. Input Parsing (main.py)
 PTT’s _safe_read_csv and _flush_pipe_block split multi-line cells containing Markdown-style lists (e.g. 1., 2.) into separate rows. Pandas treats those as new records instead of line continuations. You saw “2.” “3.” “4.” aligned while “1.” collapsed — exactly matches behavior when first line parsed as a separate row with only one populated cell. Implemented _coalesce_multiline_rows(df) to merge rows with one non-empty column into the previous cell. Fix confirmed.
2. Escaping Order (html_renderer.py)
 PTT escapes content before detecting list patterns. This converts 1. to literal text, blocking list rendering. In TV2.1, escaping happens after line/markup conversion, preserving list semantics. Move escape step after newline and list pattern handling.
3. Normalization of Newlines
 PTT normalizes only \n. Windows-originating text uses \r\n, causing occasional skipped numbering. Reproducible on exported Excel→CSV files. Normalize \r\n and \r to \n before parsing (done in benchmark).
4. HTML Conversion Path
 render_table() in PTT builds <td>s sequentially without <ol><li> wrapping when numeric patterns appear inside text cells. TV2.1 correctly identifies numbered patterns and inserts semantic <ol> tags. Add numeric detection regex (r'^\s*\d+\.') to wrap lines in <ol><li> tags for consistency.
5. Renderer Safety Layer
 PTT’s renderer attempts to re-minify or sanitize HTML after injecting list markup, occasionally flattening <li> spacing. Seen as collapsed “1.” text inside cell with no indentation. Disable post-minifier on protected sections <ol>, <li>, <pre>, <code>.
6. Table Alignment CSS/JS
 Misalignment not from CSS. Verified table grid and colgroup identical to TV2.1. DOM inspection shows correct widths; problem originates in HTML generation. No CSS change needed.
7. Caching/Interference Layer
 PTT template caching may reuse a rendered table from prior preview session. Some reloaded tables showed intermittent “1.” missing even after parser fix. Flush Flask/Jinja2 template cache before testing; add cache-busting param to render endpoint.
8. Diagnostic Repro Step
 Verified by feeding controlled multiline input:

Ultra-Detailed Comparative Technical Analysis: PTT vs TV2.1
System Role
 PTT: The operational application frame of the PasteToTable ecosystem. It runs end-to-end control: input → normalization → conversion → rendering → export. It connects browser UI, worker threads, and the Flask backend (server.py). It holds orchestrators, diagnostics, and multi-table lifecycle control.
TV2.1: The final rendered document. It is an autonomous viewer built from the render output pipeline (TV 2.1 standard). It does not accept input or convert data. Its sole function is to expose an immutable, exportable presentation layer for previously processed data.
Render Context
 PTT: Generated dynamically by Flask’s render_template(). Variables ({{version}}, {{manifest}}, {{asset_hash}}) are injected at runtime. The file changes on each build.
TV2.1: Static HTML artifact emitted by render_page() from the Python renderer. Every asset link includes versioned cache-busting hashes. It is immutable and reproducible byte-for-byte for integrity checking.
Page Skeleton
 PTT: <!DOCTYPE html><html><head>…<body> includes explicit <header>, <main>, <footer>, <aside> and separate regions for paste, preview, and export. Each section has ID hooks used by script.core.js.
TV2.1: Simplified skeleton with a single root container #tables-viewer and sub-divisions #stickyMainHeader, #tocBar, .table-wrapper. No semantic multi-region layout.
Style Layer (CSS)
 PTT: Inline <style> defines core CSS variables (--fg, --bg, --accent, --muted, --main-header-height, etc.), grid layout, responsive breakpoints, theme transitions, scrollbars, and accessibility focus outlines. External styles style.css, overrides.css, dark.css are appended conditionally.
TV2.1: Inline style limited to .table-wrapper{opacity:0;min-height:24px;transition:opacity .12s linear} for fade-in effect, then imports global CSS from assets/style.css and assets/overrides.css.
DOM Composition
 PTT: Builds DOM nodes programmatically after paste. Each table is wrapped in .table-wrapper[data-table-id]. script.table.js injects table header controls and binds events.
TV2.1: DOM already baked. The static table is pre-injected with <thead>, <tbody>, and all <tr>/<td> populated. Controls exist at top level and in each .table-header-wrapper.
JavaScript Stack
 PTT: Modular, ES5 compatible, split into six logical bundles:
• script.core.js → event bus, input handling, worker orchestration.
• script.table.js → per-table manipulation, collapse, search highlight, sorting.
• script.toc.js → TOC auto-generation.
• script.tweaks.js → UI cosmetic fixes.
• worker.js → runs Pandas parser and Markdown normalizer in a Web Worker.
• xlsx.full.min.js → third-party library for XLSX export.
TV2.1: Minimal stack: xlsx.full.min.js, script.js (bundled all minor utilities and handlers), and worker.js. No modular separation; all functions defined inside IIFEs or window scope.
Execution Order
 PTT: (1) HTML loads, (2) DOMContentLoaded triggers initPTT(), (3) event routers attach, (4) workers created, (5) ready event fired for UI activation, (6) TOC builder runs.
TV2.1: (1) Page parsed, (2) inline IIFE ensureDelegation() runs to alias hidden buttons, (3) window.load fades tables to visible. No additional setup.
Data Flow
 PTT: Input → JS → Worker → Flask /render → HTML fragment → DOM insertion → optional file export. It uses message passing (postMessage) for data exchange.
TV2.1: No data ingestion. The HTML already contains the dataset. Only export actions trigger new threads (PDF, XLSX).
Worker Usage
 PTT: Workers parse and sanitize pasted text; enforce content-length limit (MAX_PASTE_SIZE); convert to JSON, send to backend if needed.
TV2.1: Worker only handles export serialization. It never parses or mutates the DOM.
Search Logic
 PTT: Search bar connects to a debounced listener that queries all visible tables, hides non-matching rows, and highlights matches using <mark>. Supports reset button and keyboard focus cycling.
TV2.1: Search bar exists but uses a simplified inline filter operating on one table. It lacks incremental search and highlight clearing.
Collapse/Expand Mechanism
 PTT: Implements global and per-table collapse states persisted in sessionStorage. Each .toggle-table-btn toggles row visibility by adding/removing .collapsed. toggleAllTables() iterates over all wrappers.
TV2.1: Provides single global “Collapse All Tables” and per-table “Collapse Table”. No persistence; effect only lasts until reload.
Copy & Export Controls
 PTT: copyTablePlain(), copyTableMarkdown(), exportTableCSV/JSON/XLSX/PDF/Markdown() call backend endpoints for large data. Buttons dynamically show/hide depending on state.
TV2.1: Identical functions but executed locally; export uses browser’s Blob API and in-page worker. Markdown and Plain copy rely on DOM text extraction only.
Alias Delegation Script
 Both contain identical defensive IIFE ensuring that hidden #copyAllMdBtn delegates its listeners to visible #copyAllTablesBtn. It re-maps addEventListener, removeEventListener, onclick, focus, and blur to guarantee backward compatibility with older scripts that reference the alias.
Error Handling
 PTT: Each async segment wrapped in try/catch with fallback alert() plus console logging. Manifest debug logs to console.info('manifest check ok').
TV2.1: Only alerts when export worker fails or when html.length > 2 000 000. No structured diagnostic area.
Security Hardening
 PTT: Adds headers through Flask after_request middleware (X-Content-Type-Options, X-Frame-Options, Referrer-Policy). Sanitizes pasted HTML with html.escape. Validates content length before processing.
TV2.1: No dynamic input; risk surface negligible; no headers because it is a static file.
Persistence & Storage
 PTT: Theme and collapse states in localStorage / sessionStorage. Conversion results are volatile.
TV2.1: No storage interaction. State reset on reload.
Accessibility (A11y)
 PTT: ARIA roles (role="region", aria-label on controls), tab order management, keyboard shortcuts (Ctrl+F to focus search, Ctrl+E to export).
TV2.1: Basic ARIA tags only on navigation (role="navigation") and tables. Minimal keyboard affordance.
Performance & Rendering Strategy
 PTT: Virtualizes large tables via incremental DOM chunking and visibility observers to avoid lag. Lazy-renders rows beyond viewport.
TV2.1: Renders entire table statically. Suitable only for ≤ 2 000 rows. No virtualization.
Caching
 PTT: Service worker pre-caches scripts and CSS. Manifest hashed with timestamp for version invalidation.
TV2.1: No service worker; relies on browser cache of asset URLs with ?v= parameters.
Print and PDF Behavior
 PTT: Print stylesheet hides toolbars and uses compact font metrics. Export to PDF server-side uses reportlab.
TV2.1: In-browser PDF export handled by worker.js using html2pdf.js logic merged into custom worker.

 PTT: Footer embeds App v2.1.<build> with checksum. JS fetches /manifest.json for diagnostic.
TV2.1: No footer but asset querystrings (?v=1759748863) carry the hash, enabling cross-validation with source manifest.
Backend Coupling
 PTT: Coupled to Python layer for heavy processing. Dependent on /api/render, /api/export.
TV2.1: Fully decoupled; static HTML served via any HTTP server or file protocol.
Runtime Footprint
 PTT: ~380 KB total (HTML + JS + CSS). Memory footprint up to 40 MB for large tables.
TV2.1: ~110 KB total; memory ≤ 8 MB typical.
Diagnostic Channels
 PTT: Includes PTT.log() wrapper writing to hidden debug area. Logs worker lifecycle, parse time, render time.
TV2.1: Only uses console.warn() in try/catch for alias delegation and worker failures.
Testing Hooks
 PTT: Exposes window.__PTT_TEST__ API for automated Selenium/Playwright testing.

Security Edge-Cases

TV2.1: Not applicable; no input pipeline.
Interoperability
 PTT: Generates TV2.1 artifacts via render_page(). Downstream viewers, CI tests, and offline readers rely on that format.
TV2.1: Conforms to canonical TV 2.1 schema so any viewer can read it uniformly.
Maintainability
 PTT: Modular and extensible but more complex to debug. Changes ripple across worker and backend.
TV2.1: Simple to audit and diff. Used as regression baseline.
Build Pipeline Position
 PTT: Pre-export stage (“authoring environment”).
TV2.1: Post-export stage (“distribution artifact”).
Relationship Summary

 PTT: <header> (branding, instructions, buttons) → <main> (input text area, preview area) → <div id="table-wrapper"> → <footer> (version, copyright, debug manifest). Multiple invisible utility divs exist for modals, logs, and drop zones.
TV2.1: <div class="header"> + <div id="table-wrapper"> with pre-rendered <table> and static <ul class="toc">. No footer, modal, or drop zone elements.

 PTT defines viewport, charset, theme-color, manifest link, and a comprehensive inline <style> defining custom CSS variables and dark/light themes. It includes <link rel="manifest"> and <meta name="color-scheme" content="light dark">.
TV2.1 only declares basic <meta charset="utf-8"> and <link rel="stylesheet" href="style.css">. It relies on external CSS.
CSS and Theming System
 PTT defines internal variables --bg, --fg, --accent, --muted, etc. It applies a responsive grid, smooth transitions, TOC floating logic, dark/light toggling, and focus states for accessibility. Includes media queries for prefers-color-scheme.
TV2.1 defines minimal CSS inline (#table-wrapper{opacity:0;transition:opacity .2s}) then loads external CSS for final layout. It contains only essential selectors; no theme variables inline.
Scripts and Dependency Graph
 PTT loads xlsx.full.min.js, script.toc.js, script.core.js, script.table.js, script.tweaks.js, worker.js. Each script is modular: core handles paste and parsing; table manages rendering; tweaks handle alias buttons and keyboard shortcuts; toc builds navigation. Script load order is enforced by sequential <script> tags.
TV2.1 loads only xlsx.full.min.js, script.js, and worker.js. script.js merges reduced logic from all modules into a lightweight client-side renderer and export handler.
Application Logic Layers
 PTT: Three logical layers—Input (capture and normalize pasted data), Transform (send to worker, receive normalized table JSON), Output (render to DOM, build TOC, enable export). Each layer communicates asynchronously.
TV2.1: Single layer—Display and Export. Data is static; user can copy, collapse, reset, or export.
Initialization Flow
 PTT executes multi-step load: register service worker → initialize alias map → attach event delegation → run TOC build → render any cached table → ready flag. Includes recovery for failed worker startup.
TV2.1: Minimal initialization; window.onload sets opacity=1 on #table-wrapper, binds buttons, and prepares export worker. No progressive setup or cache recovery.
Table of Contents (TOC)
 PTT builds TOC dynamically using script.toc.js by scanning <h1>–<h3> tags, creating hierarchical <ul> lists with smooth scroll. It updates automatically when new tables render.
TV2.1 includes static HTML TOC with one entry per table. No dynamic update or scroll-sync.
Worker Communication Pattern
 PTT uses bidirectional postMessage with worker.js. The worker parses input (CSV/TSV/Markdown/HTML), normalizes, and sends structured table objects back. Main thread then calls render_table(). It maintains heartbeat and timeout logic.
TV2.1 uses the same worker.js but only for PDF export. One-way communication (main → worker → blob → download).
Data Flow Diagram
 PTT: User input → JS parser → worker (normalization) → render → TOC update → export.
TV2.1: Static HTML → user click → client export → done.
Export Capabilities
 PTT: Supports copy/export to Markdown, CSV, TSV, Excel (via xlsx), JSON, and PDF (via worker). Triggers backend for large exports. Buttons call unified PTTExport.dispatch().
TV2.1: Exports Markdown, CSV, PDF locally. Excel and JSON omitted to keep offline footprint small.
Alias Delegation System
 Both share identical alias mapping logic (same signature): query by [data-alias] attributes, map alias → actual element, then perform bound action. PTT has aliases for live controls (convert, paste, reset, theme). TV2.1 has reduced set (copy, collapse, reset, export).
Accessibility (ARIA)
 PTT explicitly sets role="region", aria-label="Table Output", and ensures keyboard navigability through tabindex.
