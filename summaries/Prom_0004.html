ARRANGEMENT RULES
Open with the container slides first: title, agenda, section divider, chapter opener.
Place the core narrative next: context, diagnosis, evidence, options, recommendation.
Then move to execution: plan, timeline, ownership, risks, governance.
End with support: appendix, glossary, sources, backup, notes.

CLASSIFICATION RULE
If a slide has two roles, assign it by dominant function.
Example: a chart used to support a decision is a decision slide with a chart, not a chart slide with a decision note.

CANONICAL MAPPING RULES
Use one axis for the thing itself, one for how it is delivered, and one for how it is tracked.
If a value describes what the content is, put it under Intrinsic Content.
If a value describes the slide or artifact form, put it under Delivery / Container.
If a value describes status, ownership, scope, or lifecycle, put it under Metadata / State.
Do not mix source types, evidence qualifiers, and provenance labels in the same axis.
Do not mix slide-type nouns with representation modes.
Do not mix lifecycle labels with temporal descriptors.
Do not mix audience scope with rhetorical tone.
When a term can fit more than one axis, assign it to the most structural axis and retain the rest as secondary tags.
If a label is both content and container, classify the dominant role first and the embedding role second.
If a label is both evidence and source, separate source type from confidence / corroboration.
If a label is both active-state and current-state, keep current in temporal state and active in lifecycle state.
If a label is both transcript and text, keep transcript as delivery/container and text as representation mode.
